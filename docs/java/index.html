Java 教程 · 面向 C/C++ 程序员

本教程假设你已经熟悉 C/C++(指针、栈/堆、值传递与引用、编译链接、手动内存管理)。 我们不停留在”什么是类、什么是循环”,而是聚焦:每个操作在 JVM 里到底发生了什么—— 例如 a = b 对基本类型和引用类型分别意味着什么?对象在堆上的内存布局(对象头、markword)长什么样? ArrayList、HashMap 内部结构如何?JIT、GC、happens-before 又是怎么回事?

阅读顺序(建议)

全教程贯穿的几个”底层视角”

  1. 区分”基本类型”和”引用类型”是理解 Java 的钥匙:int 直接把值放在栈帧的局部变量表里;对象永远在堆上,变量里存的是一个”指向堆的引用值”(可理解为受管理的指针)。
  2. Java 只有一种传参方式:按值传递——只不过引用类型传递的是”引用这个值”的副本,所以能改对象内容但改不了调用方的变量指向。
  3. JVM 帮你做了 C/C++ 里手动做的事:内存分配、回收(GC)、越界检查、类型安全,但代价与行为都可观测、可调优。
