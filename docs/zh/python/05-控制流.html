控制流

控制流的关键字看起来和 C 差不多:if/while/for。但底层机制差得远。C 的 for (int i=0; i<n; i++) 是编译期生成的整数递增 + 比较跳转;Python 的 for 是运行时的方法调用协议,每一次循环都在向一个对象发消息(__next__)。本章每个结构都先讲”怎么写、怎么用”——不需要懂底层就能看懂;再进入”底层与 C 对比”——看这些”看似熟悉”的结构底层到底在做什么。 只想会用,读每节前半段即可。

一、先会用:控制流速览

不需要任何底层知识,先看这几种结构怎么写:

# if / elif / else
age = 20
if age < 13:
    print("儿童")
elif age < 18:
    print("青少年")
else:
    print("成年人")

# while 循环
n = 3
while n > 0:
    print(n)
    n -= 1

# for 循环:遍历一个列表
for fruit in ["苹果", "香蕉", "橙子"]:
    print(fruit)

# break / continue
for i in range(5):
    if i == 3:
        break
    print(i)

输出:

成年人
3
2
1
苹果
香蕉
橙子
0
1
2

if/elif/else 靠缩进分支,不用括号;while 条件: 条件为真就一直循环;for 变量 in 序列: 依次把序列里每个元素赋给变量;break 提前跳出循环,continue(没在上面演示,作用是跳过本次剩余代码直接进入下一轮)。这些够写基本逻辑了。下面进入每个结构的底层机制。

二、开始深入:真值测试不止 0 和 1

C 里 if (x) 就是判断 x != 0(整数、指针都一样)。Python 里 if x: 触发的是一套协议,任何对象都能定义自己”真不真”。

怎么用

先看哪些值在 if 里被当”假”、哪些被当”真”:

if []:
    print("非空")
else:
    print("空列表是假")     # 空列表是假

if 0:
    print("非零")
else:
    print("0 是假")         # 0 是假

if "hi":
    print("非空字符串是真")  # 非空字符串是真

内置的假值一览:

假值 原因
None 单例,表示”无值”
False 就是它自己
0、0.0、0j 数值零
""、b"" 空字符串/空字节串
[]、()、{}、set() 空容器

除了上表,其余一切对象在 if 里都是”真”。这解释了为什么可以直接 if items: 判断列表非空,不用写 len(items) == 0。

底层与 C 对比

CPython 求真值的顺序(见 object.c 里的 PyObject_IsTrue):

  1. 若对象定义了 __bool__,调用它,必须返回 bool;
  2. 否则若定义了 __len__,调用它,len == 0 为假,否则为真;
  3. 都没有,则对象恒为真。
class Buffer:
    def __init__(self, data):
        self._data = data
    def __len__(self):
        # 没有 __bool__,于是 if 走 __len__
        return len(self._data)

buf = Buffer([])
print(bool(buf))          # False —— 长度为 0
print(bool(Buffer([1])))  # True

class Always:
    def __bool__(self):
        return False       # __bool__ 优先于 __len__

print(bool(Always()))     # False

这就是为什么空容器、0、0.0、""、None 都是”假”的底层原因:它们要么 __len__ 返回 0,要么 __bool__ 返回 False。

底层提示:if x: 里没有隐式的指针判空。None 才是 Python 的”空指针”,但它是个真实的单例对象(地址固定),不是 C 里的 0 地址。判空要写 if x is None,用 is 比较身份(即 C 里的指针相等),而不是 ==。

三、if / elif / else

怎么用

语法上没有括号、靠缩进,前面「先会用」已经写过一个例子。这里补充一个 C 里没有的写法——条件表达式(三目运算符的 Python 版):

n = 5
label = "正" if n > 0 else "非正"   # 值表达式,不是语句
print(label)                        # 正

顺序跟 C 的 c ? a : b 不一样:先写”真时的值”,再写条件,最后写”假时的值”。

底层与 C 对比

值得看的是 if/elif/else 编译成什么字节码——纯跳转,和 C 一样:

import dis

def classify(n):
    if n > 0:
        return "正"
    elif n == 0:
        return "零"
    else:
        return "负"

dis.dis(classify)

你会看到 COMPARE_OP 后跟 POP_JUMP_IF_FALSE——条件为假就跳到下一个分支。elif 没有独立的字节码,它只是 else: if 的语法糖,但不增加缩进层级。

四、while 循环

怎么用

while 和 C 几乎一致:

n = 10
while n:            # 等价于 while n != 0(走上一节的真值协议)
    n -= 1
print(n)             # 0

break / continue 语义与 C 相同。Python 没有 do-while,常见替代是 while True + break。

底层与 C 对比

while 的条件判断走的就是上一节讲的真值协议(__bool__ / __len__),不是简单的机器码比较——while n: 对一个自定义对象一样有效,只要它实现了 __bool__ 或 __len__。这跟 C 里 while(p) 只能判断指针是否为 NULL 不同。

五、for 循环:与 C 的下标 for 本质不同

这是本章最重要的一节。

怎么用

最常见的写法就是”对序列里的每个元素做点什么”:

for x in [10, 20, 30]:
    print(x)             # 10 20 30(逐行)

for ch in "abc":
    print(ch)             # a b c(逐行)

for i in range(5):
    print(i)              # 0 1 2 3 4(逐行)

不需要下标、不需要知道长度,for 变量 in 序列: 就会自动依次把序列里每个元素赋给变量。

底层与 C 对比

C 的 for 是”初始化 - 判断 - 递增”三段,循环变量是你手里的整数,你主动用它去索引数组。Python 的 for 完全不是这样——它背后是迭代器协议:

for x in obj:
    ...

等价于下面这段展开(这就是 CPython 实际做的事):

_it = iter(obj)          # 调用 obj.__iter__(),拿到一个迭代器对象
while True:
    try:
        x = next(_it)    # 调用 _it.__next__()
    except StopIteration:
        break
    ...

对照两个协议方法:

方法 由谁实现 职责
__iter__() 可迭代对象(iterable) 返回一个迭代器
__next__() 迭代器(iterator) 返回下一个值,耗尽时 raise StopIteration

底层意义:循环的每一步都是一次方法调用——是消费方主动向迭代器”拉(pull)“下一个值(每步调用一次 __next__),而不是迭代器把值”推”给你。所以 Python 的 for 天生能遍历链表、树、文件行、无限序列——只要对象实现了协议,不需要下标、不需要知道长度。这也是为什么遍历时你几乎从不写索引。

手写一个迭代器,看清协议:

class Countdown:
    def __init__(self, start):
        self.start = start
    def __iter__(self):
        # 返回迭代器。这里返回一个独立对象,保证可重复遍历
        return CountdownIter(self.start)

class CountdownIter:
    def __init__(self, cur):
        self.cur = cur
    def __iter__(self):
        return self          # 迭代器自身也要是可迭代的
    def __next__(self):
        if self.cur <= 0:
            raise StopIteration   # 用异常表示"结束",不是返回哨兵值
        self.cur -= 1
        return self.cur + 1

for x in Countdown(3):
    print(x)                 # 3 2 1

注意”可迭代对象”和”迭代器”是两个角色。list 是可迭代对象但不是迭代器(它没有 __next__);iter(mylist) 返回的 list_iterator 才是迭代器,它内部持有一个 C 层的索引 it_index,每次 __next__ 递增。所以对同一个 list 可以开多个独立迭代器,各有各的游标。

lst = [10, 20, 30]
a = iter(lst)
b = iter(lst)
print(next(a), next(a))   # 10 20
print(next(b))            # 10 —— b 是独立游标

内存图:list 只有一份数据,两个迭代器各自持有指向它的引用和一个整数游标:

        list_iterator a           list_iterator b
        +-----------+             +-----------+
        | it_seq  ---> [list]     | it_seq  ---> (同一 list)
        | it_index=2|      ^      | it_index=1|
        +-----------+      |      +-----------+
                           |
        list 对象  +-------+-------+
                   | ob[0]=10 (ptr)|
                   | ob[1]=20 (ptr)|
                   | ob[2]=30 (ptr)|
                   +---------------+

iter 还有个鲜为人知的双参数形式 iter(callable, sentinel):它反复调用无参的 callable,直到返回值等于 sentinel 就停(抛 StopIteration)。这正是 C 里 while ((c = getchar()) != EOF) 那种「反复取直到哨兵」惯用法的直接对应,常用于按块读流:

import io
stream = io.StringIO("a\nb\nc\n")
# 反复调用 stream.readline,直到它返回 ""(文件末尾的哨兵)才停
for line in iter(stream.readline, ""):
    print(repr(line))
# 'a\n'
# 'b\n'
# 'c\n'

它把「循环取值 + 判断结束」压缩成一个迭代器,不需要写 while True + break。真实文件也能这么用:iter(f.readline, "")、iter(lambda: sock.recv(4096), b"")。

六、range:惰性对象,不是数组

怎么用

range 用来生成一串等差整数,最常见就是配合 for 循环计数:

for i in range(5):        # 0,1,2,3,4
    print(i)

print(list(range(5)))          # [0, 1, 2, 3, 4]
print(list(range(1, 10, 2)))   # [1, 3, 5, 7, 9] —— 起点、终点(不含)、步长

底层与 C 对比

C 程序员容易把 range(1000000) 想成”生成一个百万元素的数组”。不是。range 是一个不可变的惰性序列对象,只存 start/stop/step 三个整数,几乎不占内存:

import sys
r = range(1_000_000)
print(sys.getsizeof(r))   # 48 字节左右,与元素个数无关
print(r[500])             # 500 —— 按需 O(1) 计算,不查表
print(len(r))             # 1000000 —— 由公式算出

它支持索引和 len(实现了序列协议),但值是用 start + i*step 现算的,从不物化成列表。这与 C 里”必须先分配数组再遍历”截然不同。要真正物化才用 list(range(...))。

正因为一切靠公式算,range 的切片返回的还是 range(依旧惰性),而 x in range(...) 也是 O(1) 的算术判断,不遍历十亿个元素:

r = range(0, 100, 7)
print(r[-1])                  # 98 —— 负索引也按公式算
sub = r[2:5]                  # 切片不物化,返回新的 range
print(sub)                    # range(14, 35, 7)
print(list(sub))              # [14, 21, 28]
print(70 in range(0, 1_000_000_000, 7))   # True —— 直接算 70 是否落在等差数列上,瞬间返回
print(71 in range(0, 1_000_000_000, 7))   # False —— 71 不是 7 的倍数

对比:70 in [0, 7, 14, ...](真的列表)得线性扫描;range 用「是否满足 start ≤ x < stop 且 (x-start) % step == 0」一步判完。

七、推导式:有独立作用域

怎么用

推导式是”用一行循环造一个新容器”的简写:

squares = [x * x for x in range(5)]
print(squares)                       # [0, 1, 4, 9, 16]

evens = [x for x in range(10) if x % 2 == 0]   # 带过滤条件
print(evens)                                    # [0, 2, 4, 6, 8]

square_map = {x: x * x for x in range(4)}       # 字典推导式
print(square_map)                                # {0: 0, 1: 1, 2: 4, 3: 9}

比起手写:

squares = []
for x in range(5):
    squares.append(x * x)

推导式更短,也是更地道(Pythonic)的写法。

底层与 C 对比

推导式不只是语法糖:它有自己的作用域,循环变量不会泄漏到外层。

i = "外层"
squares = [i for i in range(3)]
print(i)          # "外层" —— 循环变量 i 不泄漏到外层!

而”独立作用域”具体怎么实现,分版本:

  • Python ≤ 3.11:把推导式体编译成一个匿名函数并立即调用——反汇编能看到 MAKE_FUNCTION + CALL 和一个名为 <listcomp> 的内层 code 对象,靠独立栈帧隔离变量。
  • Python 3.12+(PEP 709):list/dict/set 推导式已内联(inlined)到当前函数里,不再生成匿名函数、也没有独立栈帧或 <listcomp> code 对象。dis 里看到的是 LOAD_FAST_AND_CLEAR + FOR_ITER + LIST_APPEND:进入推导式时把外层同名变量”借走并清空”、结束后再恢复它,从而既内联(更快)又不泄漏。
import dis
def make():
    return [i * 2 for i in range(3)]
dis.dis(make)     # 3.12+:无 MAKE_FUNCTION、无 <listcomp>;3.11 及更早才有

这与 C 的块作用域不同,也与 Python 2 不同(那时推导式变量会泄漏)。

注意:普通 for 语句没有独立作用域,循环变量会留在当前作用域。只有推导式和生成器表达式才有。

生成器表达式更进一步,连结果列表都不物化,返回一个迭代器,惰性产出(注意:生成器表达式并没有被内联,它仍编译成一个独立的 <genexpr> 函数,这也是它天然拥有独立作用域的原因):

gen = (i * 2 for i in range(1_000_000))   # 不计算任何值,不分配百万列表
print(next(gen))   # 0 —— 此刻才算第一个

八、循环的 else 子句

怎么用

Python 独有:for/while 可带 else,它在循环正常耗尽(没有被 break 打断)时执行。理解成”没 break 就走 else”:

def find(seq, target):
    for x in seq:
        if x == target:
            print("找到", x)
            break
    else:
        # 只有循环跑完都没 break 才到这里
        print("没找到")

find([1, 2, 3], 2)   # 找到 2
find([1, 2, 3], 9)   # 没找到

这省掉了 C 里常见的”设一个 found 标志变量再判断”的写法。

底层与 C 对比

底层上,break 会跳过 else 块对应的字节码,正常结束则顺序落入 else——不是通过一个隐藏的布尔标志变量实现,而是编译期就安排好了跳转目标。

九、match-case:结构化模式匹配(3.10+)

这一节我们从零讲清楚 match,因为它是 Python 里最容易看着眼熟、其实和你想的不一样的语法。

一句话:match 不是 C 的 switch。C 的 switch(x) 是”拿 x 的整数值去跳转表里查”,只能比整数、还要 break 防贯穿;Python 的 match 是”看看这个值长什么结构,并顺手把里面的部件取出来绑定到变量”。所以它更像”拆快递”:先看盒子形状对不对,对了就把里面的东西掏出来给你。

它没有 fall-through(不会像 C 那样漏写 break 就往下贯穿),也就不需要 break:匹配上哪个 case 就执行哪个,然后整个 match 结束。

第 1 步:最简单的用法——按”值”匹配(这部分最像 switch)

def http(code):
    match code:               # 拿 code 去和下面每个 case 比
        case 200:             # 如果 code == 200
            return "OK"
        case 404:
            return "Not Found"
        case _:               # _ 是通配符,谁都匹配,相当于 switch 的 default
            return "Unknown"

print(http(200))   # OK
print(http(404))   # Not Found
print(http(500))   # Unknown
  • case 200: 这种写字面量(数字、字符串、True/False/None)时,就是”值相等就匹配”,和 switch 直觉一致。
  • case _: 的下划线 _ 是通配符:放最后当兜底,类似 default。
  • 从上往下第一个匹配的 case 生效,后面的不再看。

第 2 步:OR 模式——一个 case 匹配多个值

用 | 把多个模式并起来,任意一个中就算匹配(比 C 里堆一串 case 400: case 401: 干净):

def kind(code):
    match code:
        case 200 | 201 | 204:      # 这三个值任意一个都算 success
            return "success"
        case 400 | 401 | 404:
            return "client error"
        case _:
            return "other"

print(kind(201))   # success
print(kind(404))   # client error

第 3 步:捕获模式——一边匹配一边”取值”(这是 match 的精髓)

到这里才和 switch 拉开差距。你可以在模式里写一个变量名,匹配成功时它会把对应的值绑定(赋值)给这个变量,后面直接用:

def head_tail(seq):
    match seq:
        case []:                       # 匹配空列表
            return "空的"
        case [x]:                      # 匹配"恰好一个元素的列表",并把那个元素绑定到 x
            return f"只有一个:{x}"
        case [first, *rest]:           # 匹配"至少一个元素";first=第一个,*rest=剩下的(列表)
            return f"第一个={first},剩下={rest}"

print(head_tail([]))          # 空的
print(head_tail([9]))         # 只有一个:9
print(head_tail([1, 2, 3, 4]))# 第一个=1,剩下=[2, 3, 4]
  • [x]、[first, *rest] 这种叫序列模式:它先检查”是不是序列、长度对不对”,对了再把元素拆出来绑定到变量。*rest 像函数里的 *args,收集剩余元素。
  • 元组也一样:case (x, y): 匹配”两个元素的元组”并把两个元素绑定到 x、y。

⚠️ 第 4 步:新手最大的坑——裸变量名是”绑定”不是”比较”

这是 match 唯一反直觉、几乎人人踩的点,单独讲清楚。

你可能想”如果 n 等于外面那个变量 value 就匹配”,于是写 case value:。错! 模式里的裸名字永远是”捕获绑定”,不是拿它的值去比较——它会无条件匹配,并把 n 赋给一个新的 value,还把外层同名变量盖掉:

value = 3
def bad(n):
    match n:
        case value:                    # 陷阱!这不是"n == 3",而是"把 n 绑定到 value"
            return f"绑定了 value={value}"

print(bad(99))   # 绑定了 value=99   ← 证明它根本没和外层的 3 比较,什么都能匹配

想拿一个变量的值来匹配,有两种正确写法:

value = 3
def good(n):
    match n:
        case v if v == value:          # 写法一:先捕获成 v,再用 guard 比较
            return "等于 value"
        case _:
            return "不等"

# 写法二:用"带点的名字"(限定名),点号告诉 match 这是常量引用、不是绑定
import enum
class Color(enum.Enum):
    RED = 1; GREEN = 2
def name(c):
    match c:
        case Color.RED:                # 带点 → 当作值来比较(而不是绑定)
            return "红"
        case Color.GREEN:
            return "绿"

记住一句话:模式里 abc 是”绑定”,X.abc 才是”比较”。

第 5 步:guard——给 case 加一个额外条件

在 case 后面加 if 条件,叫 guard(守卫):只有”模式匹配 且 条件为真”才算中。

def sign(n):
    match n:
        case 0:
            return "零"
        case x if x > 0:        # 先捕获成 x,再要求 x > 0
            return "正"
        case _:
            return "负"

print(sign(0), sign(5), sign(-2))   # 零 正 负

第 6 步:映射模式(匹配 dict)与类模式(匹配对象)

match 还能拆字典和自定义对象——这在处理 JSON、AST、事件时非常好用。

映射模式:只关心”有没有某些键、值是什么”,多余的键忽略:

def route(req):
    match req:
        case {"method": "GET", "path": p}:   # 要求有 method=="GET",并把 path 的值绑定到 p
            return f"GET {p}"
        case {"method": "POST"}:             # 只要求有 method=="POST",别的键不管
            return "POST"
        case _:
            return "未知请求"

print(route({"method": "GET", "path": "/users"}))   # GET /users

类模式:按类型 + 属性解构对象。给类加一个 __match_args__,就能像元组一样按位置写:

class Point:
    __match_args__ = ("x", "y")     # 声明:位置匹配时,第1个对应 x、第2个对应 y
    def __init__(self, x, y):
        self.x, self.y = x, y

def quadrant(p):
    match p:
        case Point(0, 0):                    # 位置匹配:x==0 且 y==0
            return "原点"
        case Point(x=0, y=yy):               # 关键字匹配:x==0,并把 y 绑定到 yy
            return f"在 y 轴,y={yy}"
        case Point(x, y) if x > 0 and y > 0: # 位置捕获 + guard
            return "第一象限"
        case Point():                        # 只要是 Point 就行
            return "其他 Point"
        case _:
            return "根本不是 Point"

print(quadrant(Point(0, 0)))   # 原点
print(quadrant(Point(0, 7)))   # 在 y 轴,y=7
print(quadrant(Point(1, 2)))   # 第一象限

小抄:常见模式一览

模式 例子 含义
字面量 case 200: 值相等
通配符 case _: 匹配任何东西(兜底)
捕获 case x: 总匹配,并把值绑定到 x
OR case 1 | 2 任一子模式匹配
序列 case [a, b, *rest]: 是序列且解构
映射 case {"k": v}: 是字典且含键 k
类 case Point(x, y): 是该类型且按属性解构
guard case x if x > 0: 模式匹配且条件成立
限定名比较 case Color.RED: 拿 Color.RED 的值去比较(非绑定)

底层:它到底做了什么

match 由 CPython 编译成一连串专门的匹配字节码(MATCH_SEQUENCE、MATCH_MAPPING、MATCH_CLASS 等)加跳转;匹配过程可能会调用 __len__、isinstance、按 key/属性取值等。所以它比 C 的 switch(一条跳转表指令)重得多——但换来的是对结构的表达力,这是 switch 给不了的。用 import dis; dis.dis(describe) 可以看到这些指令。

一句话总结 match:字面量按值比,裸名字是绑定,| 是或,[]/{}/类() 按结构拆,if 加条件,_ 兜底。

小结

  • 用法层面:if/elif/else 靠缩进分支,条件表达式是 a if c else b;while/for 语法接近 C,for 变量 in 序列: 不需要下标;推导式 [表达式 for x in 序列 if 条件] 是造容器的简写;for/while 可带 else(没 break 才执行);match-case 按字面量/结构匹配并解构绑定,_ 兜底、X.abc 才是比较。
  • 底层层面:
    • if/while 的条件走真值协议(__bool__ → __len__ → 恒真),判空用 is None。
    • for 是迭代器协议(__iter__/__next__/StopIteration)的语法糖,值是消费方反复调用 __next__ 主动”拉(pull)“来的,和 C 下标 for 本质不同。
    • range 是惰性对象,只存三个整数;推导式有独立作用域(3.12+ 内联实现、不再生成匿名函数),循环变量不泄漏;生成器表达式连结果都惰性化。
    • match-case 编译成专门的匹配字节码(MATCH_SEQUENCE/MATCH_MAPPING/MATCH_CLASS),是结构化解构而非整数跳转表。
