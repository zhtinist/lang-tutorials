Python 教程 · 面向 C/C++ 程序员

本教程假设你已经熟悉 C/C++(指针、栈/堆、值传递与引用、编译链接、手动内存管理)。 因此我们不重复”什么是变量、什么是循环”,而是把重点放在:每一个操作在底层到底发生了什么—— 例如 a = b 时,a 拿到的是对象的引用(指针)还是新分配了一块内存?可变对象与不可变对象在内存里有何区别? list、dict 内部是怎么实现的?GIL 为什么存在?

全教程贯穿的几个”底层视角”

  1. CPython 里没有”栈上的对象”:除了少数被解释器内部优化的情况,所有 Python 值(哪怕一个整数 3)都是堆上分配的 PyObject,变量只是指向它的名字。
  2. 变量 = 名字,不是内存槽:C 的 int a 是一块 4 字节内存;Python 的 a 是命名空间字典里的一个键,值是指向对象的指针。
  3. “可变 vs 不可变”决定一切别名行为:理解这一点就能预测所有 =、函数传参、拷贝的结果。
