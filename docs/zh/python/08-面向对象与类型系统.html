面向对象与类型系统

C++ 的对象模型是”编译期定死的内存布局 + vtable”;Python 的对象模型是”运行时可变的字典 + 描述符协议”。本章把类、实例、方法、继承、元类逐层拆开,和 C++ 对象内存布局逐一对照。 前置:见「变量与内存模型」——一切皆对象、变量即引用。

1. class 语法与实例的内存布局

class Point:
    count = 0                 # 类属性(所有实例共享)
    def __init__(self, x, y):
        self.x = x            # 实例属性,写进 self.__dict__
        self.y = y
        Point.count += 1

p = Point(3, 4)
print(p.x, p.__dict__)        # 3 {'x': 3, 'y': 4}

1.1 实例属性存在 __dict__ 里

C++ 的对象是一块连续内存,字段在编译期就有固定偏移:

C++  struct Point { int x; int y; };
p:  [ x=3 ][ y=4 ]      // 访问 p.x 就是 *(int*)(&p + 0),偏移写死在指令里

Python 默认下每个实例带一个 __dict__(一个 dict),属性名到对象引用的映射:

Python 实例 p(堆对象)
+------------------+
| ob_refcnt        |
| ob_type -> Point |
| __dict__ ------- |----> {'x': -> PyLong(3), 'y': -> PyLong(4)}
+------------------+

后果:

  • 访问 p.x 是一次字典查找,不是常数偏移解引用,更慢。
  • 你可以在运行时给任意实例增删属性:p.z = 5 只是往它的 __dict__ 塞一个键。C++ 里对象的字段集合是编译期固定的,做不到。
  • 每个实例多背一个 dict,内存开销不小。想省掉见后文 __slots__。

CPython 有个优化叫 “key-sharing dict”:同一个类的多个实例,若属性集合相同,可以共享 key 部分,只各存 value 数组,省内存。这让”实例 dict”没那么贵,但仍不如固定偏移。

2. 类也是对象,type 是元类

在 Python 里,类本身也是一个对象,它的类型是 type:

print(type(p))         # <class '__main__.Point'>   —— p 的类型是 Point
print(type(Point))     # <class 'type'>              —— Point 的类型是 type
print(type(type))      # <class 'type'>              —— type 的类型是它自己
  • Point 是一个存在于堆上的对象,你可以把它赋值、传参、放进 list。
  • 创建类的”类”叫元类(metaclass),默认是 type。class Point: ... 本质上等价于调用 type(name, bases, namespace) 动态造出一个类对象:
Point2 = type('Point2', (object,), {'x': 0})   # 和 class 语句等价

这与 C++ 完全不同:C++ 的类只是编译期概念,运行时不存在”类对象”(RTTI 的 type_info 只是只读元数据,不能当值来构造新类)。Python 的类是一等公民,可以在运行时生成、修改。元类是高级特性(ORM、框架用它做注册/校验),日常不用手写。

3. 属性查找顺序:实例 → 类 → MRO

访问 obj.attr 时,解释器按固定顺序查找。这是理解 OOP 的核心机制:

  1. 数据描述符(定义了 __set__/__delete__ 的类属性,如 property)——优先于实例 dict;
  2. 实例 __dict__——obj.__dict__['attr'];
  3. 类及其 MRO——沿 type(obj).__mro__ 逐个类查 __dict__,期间遇到非数据描述符(如普通函数)会触发描述符协议;
  4. 都没有 → 调用 __getattr__(若定义)→ 否则 AttributeError。

(细节顺序:数据描述符 > 实例 dict > 非数据描述符/类属性 > __getattr__。)

3.1 MRO 与 C3 线性化

多继承时,“类链”不是简单的深度优先,而是 C3 线性化算法算出的一个确定顺序,保证:子类在父类前、父类相对顺序保持、且唯一确定(否则报错)。

class A: pass
class B(A): pass
class C(A): pass
class D(B, C): pass

print([c.__name__ for c in D.__mro__])
# ['D', 'B', 'C', 'A', 'object']

C3 解决了 C++ 里臭名昭著的菱形继承问题。C++ 要靠 virtual 继承手工处理重复基类;Python 用 MRO 自动给出一个线性顺序,super() 就沿这个顺序走(见第 6 节)。

4. 方法 = 函数 + 描述符协议:绑定方法是怎么来的

一个”方法”其实就是定义在类 __dict__ 里的普通函数。神奇之处在于:函数是一个描述符(实现了 __get__)。

class Foo:
    def bar(self):
        return self

f = Foo()
print(Foo.bar)      # <function Foo.bar>       —— 通过类拿到的是纯函数
print(f.bar)        # <bound method Foo.bar>   —— 通过实例拿到的是"绑定方法"

发生了什么:f.bar 触发属性查找,在类里找到函数对象 bar,因为函数有 __get__,解释器调用 bar.__get__(f, Foo),它返回一个绑定方法对象——一个把 f 预先塞进第一个参数(self)的可调用包装:

f.bar()  等价于  Foo.bar(f)
         绑定方法内部保存了 (__func__ = bar, __self__ = f)
         调用时把 __self__ 作为第一个实参自动传入

这解释了为什么方法第一个参数必须显式写 self——它不是关键字,只是”绑定时被自动填入的那个参数”的约定名。对照 C++:C++ 的 this 是编译器隐式传入的隐藏指针;Python 把这件事显式化为描述符协议在运行时完成。

5. @property / @staticmethod / @classmethod

这三个都是”内置描述符”,用来改变属性/方法的绑定行为:

class Circle:
    def __init__(self, r):
        self._r = r

    @property                       # 把方法伪装成只读属性,访问时自动调用
    def area(self):                 # 数据描述符,优先级高于实例 dict
        return 3.14159 * self._r ** 2

    @staticmethod                   # 不绑定 self/cls,就是个放在类命名空间的普通函数
    def unit():
        return "cm"

    @classmethod                    # 绑定的是"类"而非实例,第一个参数是 cls
    def from_diameter(cls, d):
        return cls(d / 2)           # cls 让子类调用时返回子类实例(多态构造)

c = Circle(2)
print(c.area)                       # 6.28...  —— 像字段一样访问,底层调了方法
print(Circle.from_diameter(10)._r)  # 5.0
装饰器 第一个参数 通过实例调用时 典型用途
普通方法 self(实例) 绑定实例 常规行为
@classmethod cls(类) 绑定类 备选构造函数、操作类状态
@staticmethod 无 不绑定 逻辑归属该类但不需 self/cls
@property self 访问即调用 计算属性、只读封装、加校验

@property 就是 C++ 里”用 getter/setter 封装私有字段”的语言级支持——但对调用方透明:c.area 看起来是字段访问,不用改调用代码就能从字段升级成计算属性。

6. 继承与 super

class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return "..."

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)      # 沿 MRO 调用上一个类的实现
        self.breed = breed
    def speak(self):
        return "Woof"

super()(3 中的无参形式)不是”调父类”,而是”沿当前实例的 MRO 找下一个类”。在多继承里这至关重要——它保证协作式多继承时每个基类的方法恰好被调用一次,按 C3 顺序串起来。这比 C++ 里手写 Base::method()(硬编码具体基类)更灵活,也避免菱形继承里基类被调两次。

协作式多继承实测:菱形结构里 A.__init__ 只被调用一次,super() 严格沿 MRO [D, B, C, A] 前进:

class A:
    def __init__(self): print("A")
class B(A):
    def __init__(self): print("B"); super().__init__()
class C(A):
    def __init__(self): print("C"); super().__init__()
class D(B, C):
    def __init__(self): print("D"); super().__init__()

D()
# D
# B
# C
# A   <- 只有一次;若把 super() 换成硬编码 A.__init__(self),A 会被调两次

7. 鸭子类型 vs C++ 虚函数/静态类型

C++ 的多态靠虚函数表(vtable):对象头部藏一个 vptr 指向 vtable,虚调用是”查表 + 间接跳转”,且类型在编译期检查——只有声明了共同基类才能多态。

Python 是鸭子类型(duck typing):调用 x.speak() 时,解释器根本不关心 x 是什么类型,只在运行时去 x 的类型里查 speak 属性,查到就调。“如果它走起来像鸭子、叫起来像鸭子,那就当它是鸭子。”

class Duck:
    def speak(self): return "Quack"

class Person:
    def speak(self): return "Hello"

def make_it_speak(x):        # 不需要任何共同基类或接口声明
    return x.speak()         # 运行时按 x 的实际类型查方法

对照:

C++ 虚函数 Python 鸭子类型
分派时机 编译期定接口,运行时查 vtable 全在运行时查类型的 dict
需要共同基类 是(继承 + virtual) 否(有同名方法即可)
类型检查 编译期静态 运行时,错了才抛 AttributeError
开销 一次间接跳转 一次字典查找(更慢)

代价是”没声明接口,错误推迟到运行时”。折中方案见第 10 节的抽象基类,以及 类型注解 + mypy 做静态检查。

8. __slots__:用固定数组替代 dict

默认的实例 __dict__ 灵活但费内存。若一个类要创建海量实例(如百万个 Point),可以用 __slots__ 声明固定字段集合:

class Point:
    __slots__ = ('x', 'y')      # 声明后:不再有 __dict__,字段存在固定槽位
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(3, 4)
# p.z = 5          # AttributeError:不能再动态加属性
# p.__dict__       # AttributeError:没有 __dict__ 了

底层变化:CPython 在类型对象上为每个 slot 分配一个固定偏移的槽位(通过描述符实现),实例不再挂 dict,而是像 C struct 一样把字段值指针连续存在对象体内:

带 __dict__ 的实例:  [头][__dict__ ptr] -> {'x':.., 'y':..}   (额外一个 dict)
带 __slots__ 的实例:  [头][x ptr][y ptr]                        (字段内联,无 dict)

用 sys.getsizeof 实测两者内存差(CPython 3.14,64 位):

import sys
class WithDict:
    def __init__(self, x, y): self.x, self.y = x, y
class WithSlots:
    __slots__ = ('x', 'y')
    def __init__(self, x, y): self.x, self.y = x, y

a = WithDict(1, 2); b = WithSlots(1, 2)
print(sys.getsizeof(a) + sys.getsizeof(a.__dict__))  # 344  = 实例体 48 + __dict__ 296
print(sys.getsizeof(b))                              # 48   字段内联,无独立 dict
print(hasattr(b, "__dict__"))                        # False

收益:每个实例省掉一个 dict(几十到上百字节),访问变成固定偏移(接近 C++ 字段访问,快)。代价:失去动态增删属性、失去 __dict__、多继承时 slots 有限制。这是 Python 对象模型里最接近 C struct 内存布局的写法。

9. 魔术方法与运算符重载

“魔术方法”(dunder methods,双下划线)是解释器在特定语法/操作时回调的钩子。它们让你的对象融入语言的运算符、内置函数、协议。

class Vec:
    def __init__(self, x, y): self.x, self.y = x, y
    def __repr__(self):  return f"Vec({self.x}, {self.y})"   # repr()/交互式显示
    def __add__(self, o): return Vec(self.x+o.x, self.y+o.y) # 支持 v1 + v2
    def __eq__(self, o):  return (self.x, self.y) == (o.x, o.y)
    def __len__(self):    return 2                            # 支持 len(v)
    def __getitem__(self, i): return (self.x, self.y)[i]     # 支持 v[0]、迭代
    def __hash__(self):   return hash((self.x, self.y))      # 可放进 set/dict key

常见魔术方法:

语法/操作 魔术方法 对应 C++
a + b __add__ / __radd__ operator+
a == b __eq__ operator==
a < b __lt__ operator<
len(a) __len__ size() 约定
a[i] __getitem__/__setitem__ operator[]
a() __call__ operator()(仿函数)
str(a)/print __str__ operator<< 约定
with a: __enter__/__exit__ RAII(见第 9 章)
for x in a __iter__/__next__ 迭代器

运算符重载的能力和 C++ 相当,但全在运行时通过查方法完成,没有编译期重载决议。

10. 抽象基类(abc)

鸭子类型灵活,但有时你想强制”子类必须实现某些方法”,或想做 isinstance 归类。abc 模块提供抽象基类:

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self) -> float: ...    # 抽象方法,子类必须实现

class Square(Shape):
    def __init__(self, s): self.s = s
    def area(self): return self.s * self.s

# Shape()      # TypeError:不能实例化含未实现抽象方法的类
Square(3)      # OK
  • 类似 C++ 的纯虚函数(virtual void area() = 0;)+ 抽象类不可实例化。
  • 但检查时机是实例化时(运行时),不是编译期。
  • abc 还支持”虚拟子类”注册和结构化的 isinstance 检查;collections.abc 定义了 Iterable/Sequence/Mapping 等协议基类,可用于类型判断。
  • 更轻量的现代做法是 typing.Protocol(结构化子类型 / 静态鸭子类型),配合 mypy 在编译期(检查期)验证接口,而不侵入运行时。

小结

  • 实例属性默认存 __dict__,访问是字典查找,不是 C++ 的固定偏移;__slots__ 可换回接近 struct 的固定槽位。
  • 类是运行时对象,type 是默认元类。
  • 属性查找:数据描述符 → 实例 dict → MRO(C3 线性化)。
  • 方法是函数 + 描述符协议,self/绑定方法由 __get__ 在运行时产生。
  • 多态靠运行时查方法(鸭子类型),而非编译期 vtable;要接口约束用 abc/Protocol。
