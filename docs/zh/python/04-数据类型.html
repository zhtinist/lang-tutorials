数据类型

上一章我们建立了「变量是指向对象的引用」的世界观。本章把常用的内置数据类型逐个拆开,看它们在 CPython 里到底长什么样——PyObject 头部之后跟着什么字节,和你熟悉的 C 定长 int、IEEE-754 double、char* 差在哪里。

贯穿本章的一句话:在 C 里类型决定了变量占几个字节;在 Python 里类型是对象堆里的一块结构体,变量只是指向它的指针。 所以「一个 int 占多少字节」这个问题,在 Python 里问的是「这个 PyLongObject 结构体有多大」,而不是「机器字长」。

4.1 一切皆对象:统一的对象头

回顾 见「变量与内存模型」,CPython 中每个对象在堆上都以 PyObject(或变长的 PyVarObject)开头:

// CPython 源码 Include/object.h(简化)
typedef struct _object {
    Py_ssize_t    ob_refcnt;    // 引用计数
    PyTypeObject *ob_type;      // 指向类型对象的指针(相当于 C++ 的 vptr)
} PyObject;

typedef struct {
    PyObject ob_base;
    Py_ssize_t ob_size;         // 变长对象的"元素个数"
} PyVarObject;

ob_type 就是 C++ 里的虚表指针(vptr)的类比:它指向一个 PyTypeObject,里面存着「这个类型怎么加法、怎么打印、怎么取长度」等一堆函数指针(称为 tp_slots)。区别在于 C++ 的 vtable 在编译期定死,而 Python 的类型对象是运行期可修改的普通对象。

在 64 位 CPython 上,这个头是 16 字节(PyObject)或 24 字节(PyVarObject)。任何 Python 对象都至少这么大——这也是为什么 Python 的 int 动辄二三十字节,而 C 的 int 只有 4 字节。

import sys
print(sys.getsizeof(0))      # 28(64 位 CPython 3.12+;更早版本是 24)
print(sys.getsizeof(1.0))    # 24

4.2 int:任意精度的变长大整数

4.2.1 语义:数学意义上的整数,不会溢出

在 C 里 int32_t x = INT_MAX; x + 1; 会溢出回绕成负数。Python 的 int 是任意精度的:

x = 2 ** 100
print(x)            # 1267650600228229401496703205376,不溢出
print(x * x)        # 继续放大也没问题

这不是魔法,而是把整数存成一个「digit 数组」,像小学竖式一样按需扩展位数——本质上就是 C 里手写的 bignum(GMP 那类库干的事),只不过内置了。

4.2.2 内存布局:PyLongObject 与 30-bit digit

CPython 的整数是变长对象 PyLongObject:

// 简化后的概念结构
typedef struct {
    PyObject_VAR_HEAD          // ob_refcnt, ob_type, ob_size
    digit ob_digit[1];         // 变长数组,每个 digit 存 30 位
} PyLongObject;

关键点:

  • 每个 digit 是一个 32 位无符号整数(uint32_t),但只用其中 30 位(PYLONG_BITS_IN_DIGIT == 30)。留出的位方便进位、借位运算不溢出 C 的 32 位。
  • ob_size 的绝对值是 digit 个数,符号表示整数正负;ob_size == 0 表示数值 0。这是「符号-数值」表示法,不是补码。

注:CPython 3.12 起内部改用 ob_size + 一个标志位打包的 lv_tag(PEP 不涉及,属实现细节),但「30-bit digit 数组 + 符号」这个模型不变。

一个 ASCII 图,表示 2**60(需要 3 个 30-bit digit,因为 60 位放不进两个满 30 位……实际上 2**60 恰好第 61 位为 1):

值 = digit[0] * 2^0 + digit[1] * 2^30 + digit[2] * 2^60

+-------------------+
| ob_refcnt         |  \
| ob_type  --> int  |   > 对象头 (PyVarObject)
| ob_size = 3       |  /   (正号 + 3 个 digit)
+-------------------+
| ob_digit[0] (30b) |
| ob_digit[1] (30b) |
| ob_digit[2] (30b) |
+-------------------+

和 C 对照:

C int32_t Python int
大小 固定 4 字节 变长,随数值增大
溢出 回绕(UB/实现定义) 不溢出,自动扩容
负数表示 二进制补码 符号位 + 数值(digit 数组)
位置 可在栈上/寄存器 永远在堆上,变量持指针
运算 单条 CPU 指令 函数调用,逐 digit 处理

用 sys.getsizeof 观察增长:

import sys
print(sys.getsizeof(0))          # 28(空 digit 情况,依版本)
print(sys.getsizeof(2**29))      # 28  —— 1 个 digit
print(sys.getsizeof(2**30))      # 32  —— 需要 2 个 digit
print(sys.getsizeof(2**60))      # 36  —— 3 个 digit
print(sys.getsizeof(2**300))     # 更大

每多一个 30-bit digit,大约多 4 字节,和上面的结构完全吻合。

4.2.3 小整数缓存(small int cache)

CPython 在启动时预先创建了 [-5, 256] 范围内的所有整数对象,永久驻留。所有对这些值的引用都指向同一个对象:

a = 100
b = 100
print(a is b)       # True —— 同一个缓存对象

x = 1000
y = 1000
print(x is y)                       # 同一编译单元里通常是 True:字面量 1000 在 co_consts 中被复用(是常量去重,不是缓存)
print(int("1000") is int("1000"))   # False:运行时各自新建,才真正落在缓存范围之外

这是纯粹的 CPython 优化(类比 C++ 编译器对小字面量的处理无关,更像是一个共享的常量池)。永远不要用 is 比较数值,用 ==。

4.2.4 dis 看整数字面量

import dis
dis.dis(compile("x = 2 + 3", "<s>", "exec"))

你会看到 LOAD_CONST 5——编译期常量折叠已经把 2 + 3 算成了 5,运行时并不做加法。这类似 C 编译器的常量折叠。

4.3 bool:int 的子类

Python 里 bool 继承自 int,True 和 False 就是值为 1 和 0 的两个单例整数对象:

print(isinstance(True, int))   # True
print(True + True)             # 2 —— 当成整数参与运算
print(True == 1)               # True
print(True is 1)               # False —— 值相等但不是同一对象

底层上 True/False 是两个预先创建、全程唯一的 PyLongObject(带 bool 类型)。这和 C 的 bool(_Bool,通常 1 字节)完全不同:C 的 bool 是栈上一个字节,Python 的 bool 是堆上一个继承自 int 的对象。

实用后果:True 可当数组下标、可求和统计布尔数量:

data = [1, -2, 3, -4, 5]
positives = sum(x > 0 for x in data)   # 把 True/False 当 1/0 求和 = 3

4.4 float:装箱的 C double

Python 的 float 就是一个 IEEE-754 双精度浮点,装箱(boxing)在对象里:

typedef struct {
    PyObject_HEAD
    double ob_fval;      // 一个货真价实的 C double
} PyFloatObject;

所以一个 float 对象 = 16 字节对象头 + 8 字节 double = 24 字节:

import sys
print(sys.getsizeof(3.14))   # 24

对照 C:C 的 double x = 3.14; 是栈上 8 字节裸值,运算直接走 FPU 指令。Python 里每个 float 是堆对象,a + b 要解引用两个指针取出 ob_fval、相加、再分配一个新 float 对象装结果。这就是数值密集计算下纯 Python 慢、且内存开销大的根源,也是 NumPy 存在的理由——NumPy 把一亿个 double 打包进一块连续 C 数组,而不是一亿个 24 字节对象(见「常用第三方库」)。

浮点的所有「坑」和 C 完全一样,因为底层就是同一个 double:

print(0.1 + 0.2)            # 0.30000000000000004
print(0.1 + 0.2 == 0.3)     # False

特殊值也一致:float('inf')、float('-inf')、float('nan'),nan != nan 成立。

需要精确十进制(金融)时用标准库 decimal.Decimal;需要精确分数用 fractions.Fraction——它们内部用整数实现,不是 double。

4.5 complex:两个 double

complex 就是一个装了两个 C double(实部、虚部)的对象:

typedef struct {
    PyObject_HEAD
    double real;
    double imag;
} PyComplexObject;
z = 3 + 4j
print(z.real, z.imag)   # 3.0 4.0
print(abs(z))           # 5.0

虚部字面量后缀是 j。这是内置类型,不需要 import。

4.6 str:PEP 393 紧凑 Unicode

字符串是 C/Python 差异最大的地方之一。C 的 char* 是一串以 \0 结尾的字节;Python 的 str 是不可变的 Unicode 码点序列,且 CPython 会根据内容选择最省的内部编码。

4.6.1 PEP 393:按最大码点选择存储宽度

从 Python 3.3 起(PEP 393),CPython 不再固定用 2 或 4 字节存每个字符,而是看字符串里最大的码点,选三种紧凑表示之一:

内部种类 每字符字节 可表示范围 例子
Latin-1 (PyUnicode_1BYTE_KIND) 1 U+0000–U+00FF "hello", "café"
UCS-2 (PyUnicode_2BYTE_KIND) 2 U+0000–U+FFFF "你好"
UCS-4 (PyUnicode_4BYTE_KIND) 4 U+0000–U+10FFFF "😀"(需要补充平面)

只要字符串里出现一个高码点字符,整个字符串升到更宽的表示。观察:

import sys
print(sys.getsizeof(""))        # 41(64 位 CPython 3.12+;3.11 及更早约 49,PEP 623 移除 wstr 后头部变小)
print(sys.getsizeof("a"))       # 42   —— +1 字节 (Latin-1)
print(sys.getsizeof("ab"))      # 43   —— +2 字节
print(sys.getsizeof("你"))      # 60 —— UCS-2,每字符 2 字节且基线更大
print(sys.getsizeof("😀"))      # 64 —— UCS-4,每字符 4 字节

内存布局(紧凑形式)概念图,"café"(全在 Latin-1 范围内):

+---------------------------+
| ob_refcnt / ob_type       |  对象头
| length = 4                |  码点个数(不是字节数)
| hash (缓存)               |
| kind = 1BYTE, flags...    |  存储种类
+---------------------------+
| 'c' 'a' 'f' 'é' '\0'      |  紧接对象之后的连续码点数据
+---------------------------+

关键差异与 C 对照:

C char* Python str
单元 字节 Unicode 码点
长度 靠 \0 结尾,O(n) 求长 显式存 length,O(1)
len("😀") 按字节数(4,UTF-8) 1(一个码点)
可变性 可改 不可变
索引 s[i] 取字节 s[i] 取码点,O(1)

len() 数的是码点不是字节,这点常让写惯 C 的人踩坑:

s = "café😀"
print(len(s))            # 5 个码点
print(len(s.encode()))   # 8 个 UTF-8 字节

4.6.2 不可变性与哈希缓存

str 不可变,任何「修改」都会产生新对象。因为不可变,字符串可以缓存哈希值(见上面布局里的 hash 字段),这让它成为高效的 dict 键(见「复合数据结构」)。

s = "hello"
# s[0] = 'H'   # TypeError: 'str' object does not support item assignment
s2 = s.replace("h", "H")   # 新对象
print(s is s2)             # False

4.6.3 字符串驻留(interning)

CPython 对看起来像标识符的短字符串字面量做 interning(驻留到全局池),以便 is 快速比较、省内存:

a = "hello"
b = "hello"
print(a is b)          # 通常 True —— 同一编译单元字面量被驻留

c = "".join(["h", "e", "l", "l", "o"])
print(a is c)          # 通常 False —— 运行期构造,未驻留
print(a == c)          # True —— 内容相等

依旧那句话:比较内容用 ==,is 只判断是否同一对象。可用 sys.intern() 手动驻留高频字符串。

4.7 bytes 与 bytearray:才是「C 的 char 数组」

如果你想要 C 里那个「一串裸字节」,对应的是 bytes(不可变)和 bytearray(可变),不是 str。

b = b"hello"          # bytes 字面量,前缀 b
print(b[0])           # 104 —— 取出的是 int(字节值),不是字符
print(type(b[0]))     # <class 'int'>
  • bytes:不可变字节序列,内部就是一段连续的 char(unsigned char)缓冲 + 长度。最接近 C 的 const char* + 长度。
  • bytearray:可变版本,内部像 C++ 的 std::vector<uint8_t>——有容量、可原地增删:
ba = bytearray(b"hi")
ba.append(33)         # 追加 '!' (0x21)
ba[0] = 72            # 原地修改 -> 'H'
print(ba)             # bytearray(b'Hi!')

str 与 bytes 的转换必须显式指定编码,这是 Python 3 强制的(Python 2 的隐式转换是历史大坑):

s = "café"
data = s.encode("utf-8")     # str -> bytes
print(data)                  # b'caf\xc3\xa9'
back = data.decode("utf-8")  # bytes -> str
print(back)                  # café

对照 C:C 里 char* 既是文本又是字节,编码全靠约定;Python 强制你在边界(文件、网络、系统调用)显式编解码,str 处理文本,bytes 处理二进制。

4.8 单例:None、NotImplemented、Ellipsis

这三个是全程唯一的单例对象,类似 C++ 里你可能定义的全局 sentinel。

None

表示「无值/空」,类比 C 的 NULL,但它是个真正的对象(类型 NoneType),不是 0 指针:

x = None
print(x is None)        # 判空的标准写法,永远用 is
print(type(None))       # <class 'NoneType'>

None 全程只有一个实例,所以用 is None 而不是 == None——既快又不会被自定义 __eq__ 干扰。函数没有 return 时默认返回 None(见「函数与调用栈」)。

NotImplemented

运算符重载里,当某类型不知道怎么和对方运算时,返回它(不是抛异常),让 Python 去尝试对方的反射方法:

class Vec:
    def __init__(self, x): self.x = x
    def __add__(self, other):
        if not isinstance(other, Vec):
            return NotImplemented    # 交给对方处理
        return Vec(self.x + other.x)

Ellipsis(...)

字面量就是三个点 ...。本身几乎无内置语义,常用作占位符,或被 NumPy 用作多维切片(arr[..., 0]):

def todo(): ...          # 合法的函数体占位(等价于 pass)
print(... is Ellipsis)   # True

4.9 数值与字符串转换

显式构造/转换用类型名当函数(其实是调用类型对象):

int("42")           # 42        —— str -> int
int("ff", 16)       # 255       —— 指定进制
int(3.9)            # 3         —— float 截断(向零取整,不是四舍五入)
float("3.14")       # 3.14
float("inf")        # inf
str(255)            # "255"
bin(255)            # "0b11111111"
hex(255)            # "0xff"
oct(255)            # "0o377"

格式化(f-string,3.6+)是最常用的数值转字符串方式:

n = 255
print(f"{n:d} {n:b} {n:x} {n:08b}")   # 255 11111111 ff 11111111
pi = 3.14159
print(f"{pi:.2f}")                    # 3.14
print(f"{1234567:,}")                 # 1,234,567 —— 千位分隔

转换失败会抛 ValueError,而不是像 C 的 atoi 那样静默返回 0:

try:
    int("abc")
except ValueError as e:
    print("转换失败:", e)

这种「出错就抛异常」的取向,是 Python 相对 C 返回错误码的一大风格差异,详见 见「错误处理与资源管理」。

小结

  • 所有类型都是堆对象,共享 PyObject 头(引用计数 + 类型指针,类型指针相当于 vptr)。
  • int 是变长 bignum(30-bit digit 数组 + 符号),不溢出;[-5,256] 有缓存单例。
  • bool 是 int 子类,True/False 是值 1/0 的单例。
  • float/complex 是装箱的 C double(一个/两个),运算都要新分配对象。
  • str 是不可变 Unicode,按最大码点选 1/2/4 字节紧凑存储(PEP 393),len 数码点;bytes/bytearray 才对应 C 的字节数组。
  • None/NotImplemented/Ellipsis 是三个全程单例,判等用 is。
  • 转换用类型名调用,失败抛 ValueError 而非静默。
