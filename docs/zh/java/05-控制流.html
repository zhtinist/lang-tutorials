控制流

控制流大部分与 C 一致，本章聚焦差异点和字节码层面的机制，不重复讲 if/for/while 的通识。重点：switch 的三种形态与其字节码、增强 for 背后的迭代器、模式匹配、文本块。

一、if

和 C 基本一样，唯一硬性差异：条件必须是 boolean（见「基础语法」）。if (n) 这种”非零即真”在 Java 编译不过，必须写 if (n != 0)。好处是根除了 if (a = b) 误写赋值的经典 bug——if (a = b) 里 a = b 若非 boolean 类型直接编译错误。

字节码上，if 就是条件比较指令（if_icmpge、ifnull、ifeq 等）加跳转，和第 1 章看到的一样，没有惊喜。

二、switch：三种形态 + 字节码

2.1 传统 switch：fall-through 陷阱依旧

和 C 一模一样：不写 break 会贯穿（fall-through）到下一个 case。

int day = 3;
switch (day) {
    case 1:
    case 2:
    case 3:
        System.out.println("工作日前半");
        break;                       // 不写就贯穿，和 C 同坑
    case 6:
    case 7:
        System.out.println("周末");
        break;
    default:
        System.out.println("其它");
}

传统 switch 支持的类型：整型（及可拆箱到 int 的包装类）、char、enum、以及 String（C 不支持 switch 字符串）。String 的 switch 由编译器降级为”先比 hashCode 定位再 equals 确认”两级判断实现。

2.2 表达式 switch（->，JDK14+）：无贯穿、可返回值

新语法用 ->，不贯穿、无需 break，还能作为表达式求值：

int day = 3;
String kind = switch (day) {
    case 1, 2, 3, 4, 5 -> "工作日";       // 多标签逗号并列
    case 6, 7          -> "周末";
    default            -> "非法";
};                                        // 注意作为表达式，末尾有分号

// 需要多条语句时用块 + yield 返回值
int score = switch (grade) {
    case 'A' -> 100;
    case 'B' -> { int base = 80; yield base + 5; }  // yield 交出块的值
    default  -> 0;
};

要点： - -> 分支执行完自动跳出，没有 fall-through，从语法上消灭了忘写 break 的 bug。 - 作为表达式时，编译器要求穷尽所有可能（对 enum 覆盖全部枚举值，或提供 default），否则编译错误——比 C 更安全。 - 多语句分支用 {} 块，用 yield 交出值（return 在这里会返回整个方法，别混淆）。

2.3 模式匹配 switch（JDK21+）：按类型分派

switch 可以对类型做匹配并绑定变量，配合 sealed 类型能穷尽检查：

Object obj = ...;
String desc = switch (obj) {
    case Integer i when i > 0 -> "正整数 " + i;   // when 是守卫条件
    case Integer i            -> "非正整数 " + i;
    case String s             -> "字符串长度 " + s.length();
    case null                 -> "空值";          // 可显式匹配 null
    default                   -> "其它类型";
};

这相当于把一串 if (obj instanceof Integer i) ... else if (obj instanceof String s) ... 收拢成结构化写法，是 Java 向”代数数据类型 + 模式匹配”靠拢的产物。与 C 的 switch 完全不同层次。

2.4 字节码：tableswitch vs lookupswitch

switch 在字节码里有两条指令，编译器按 case 值的分布自动选择——这是 switch 常比等价 if-else 链更快的原因：

指令 适用 机制 复杂度
tableswitch case 值连续或接近连续 用值作跳转表索引直接跳 O(1)
lookupswitch case 值稀疏 存 (值,偏移) 有序对，二分查找 O(log n)

tableswitch 就像 C 编译器对密集 case 生成的跳转表（jump table）：把 case 1/2/3 编成一张连续偏移表，运行时 目标 = 表[值 - 低界]，一次寻址完成，无需逐个比较。lookupswitch 用于 case 1/100/10000 这种稀疏值，存排序好的键值对做二分。可用 javap -c 观察：

javap -c YourClass     // 会看到 tableswitch { ... } 或 lookupswitch { ... }

连续小整数 case → tableswitch（最快）；稀疏 → lookupswitch。这提示：性能敏感场景，让 case 值尽量连续有利于生成跳转表。

三、循环

for / while / do-while

三者与 C 语义完全一致，不赘述：

for (int i = 0; i < n; i++) { ... }
while (cond) { ... }
do { ... } while (cond);   // 至少执行一次

字节码就是循环判断 + goto 回边（见第 1 章的 goto 4）。回边计数正是 HotSpot 判定”热循环”触发 JIT 的依据之一。

增强 for（for-each）：基于 Iterable/迭代器

int[] arr = {1, 2, 3};
for (int x : arr) { ... }              // 遍历数组

List<String> list = List.of("a","b");
for (String s : list) { ... }          // 遍历集合

底层机制取决于被遍历者： - 数组：编译器降级为普通带下标的 for 循环（用 length 控制、下标访问），没有迭代器对象，零额外开销。 - 实现了 Iterable 的集合：编译器降级为调用 iterator() 取迭代器，循环 hasNext()/next()。等价展开：

// for (String s : list) {...} 实际编译成 ↓
for (Iterator<String> it = list.iterator(); it.hasNext(); ) {
    String s = it.next();
    ...
}

对照 C++ 的 range-based for（for (auto& x : container) 靠 begin()/end()）几乎一一对应：Java 靠 Iterable.iterator() + Iterator 接口。注意用增强 for 遍历集合时不能在循环体里增删该集合，否则迭代器检测到结构变化抛 ConcurrentModificationException（要删除得用迭代器自己的 remove() 或用 removeIf）。集合与迭代器详见「复合数据结构」。

四、带标签的 break / continue

Java 没有 goto（是保留字但不用），但提供带标签的 break/continue来跳出/继续外层循环，覆盖了 C 里用 goto 跳出多重循环的常见场景：

outer:
for (int i = 0; i < n; i++) {
    for (int j = 0; j < m; j++) {
        if (matrix[i][j] == target) {
            found = true;
            break outer;        // 直接跳出外层两重循环
        }
        if (skip) continue outer;  // 直接进入外层的下一轮
    }
}

break outer 结束标签所指的整个循环，continue outer 跳到该循环的下一次迭代。比 C 用 flag 变量或 goto 更结构化、更可读。字节码上仍是 goto 到相应偏移。

五、三元运算符

int max = (a > b) ? a : b;

与 C 相同。注意 Java 里三元表达式的两个分支要有兼容的类型（会做类型提升 / 装箱），比如 cond ? 1 : 2.0 整体是 double。混用包装类与基本类型时可能触发自动拆箱导致意外 NPE，留意。

六、instanceof 模式匹配

传统 instanceof 判类型后还要手动强转，模式匹配（JDK16+）一步到位并绑定变量：

// 传统写法
if (obj instanceof String) {
    String s = (String) obj;    // 还得再强转
    System.out.println(s.length());
}

// 模式匹配写法
if (obj instanceof String s) {  // 判定成功即把 obj 绑定到 s（已是 String 类型）
    System.out.println(s.length());
}

// 可在同一条件里继续用绑定变量
if (obj instanceof String s && s.length() > 3) { ... }

s 的作用域是”能确定它一定是 String 的地方”（流敏感作用域）。这与第 8 章的类型系统、以及本章的模式匹配 switch 是同一套语言演进方向。相当于把 C++ 的 if (auto* p = dynamic_cast<String*>(obj)) 做得更顺手。

七、文本块（Text Block，JDK15+）

多行字符串字面量，免去手写 \n 和拼接，类似其它语言的 heredoc：

String json = """
        {
          "name": "Ada",
          "age": 36
        }
        """;                    // 三引号包裹；编译期确定缩进基线并去掉公共前导空白

要点： - 它仍然是一个普通的 String（不可变、进常量池），只是书写方式不同，运行期与普通字符串无异。 - 编译器按结束 """ 的缩进作为基线，剥掉每行的公共前导空白，方便你在代码里对齐缩进而不污染内容。 - 适合嵌 JSON / SQL / HTML。对照 C 里只能用一堆 "..." 相邻拼接或行尾 \，文本块干净得多。

小结

  • if 条件必须 boolean，杜绝 if(a=b) 类误写。
  • switch 三形态：传统（有 fall-through）、-> 表达式（不贯穿、可返回值、要求穷尽）、模式匹配（按类型 + when 守卫）；String/enum 均可 switch。
  • 字节码 tableswitch（连续值→跳转表，O(1)）vs lookupswitch（稀疏值→二分，O(log n)），编译器自动选，可用 javap -c 查看。
  • 增强 for：数组降级为下标循环、集合靠 Iterable.iterator()；遍历中改集合抛 ConcurrentModificationException。
  • 带标签 break/continue 替代跨层 goto。
  • instanceof 模式 判类型即绑定，文本块写多行字符串。
