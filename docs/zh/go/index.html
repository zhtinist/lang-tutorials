Go 教程 · 面向 C/C++ 程序员

本教程假设你已经熟悉 C/C++(指针、栈/堆、值传递与引用、编译链接、手动内存管理)。 Go 对 C/C++ 程序员最”亲切”——它有真正的指针、值类型、结构体、编译成原生机器码; 但也有很多需要看清底层的地方:每个操作到底发生了什么—— 例如 a = b 对值类型是逐字节拷贝、对 slice/map 却是拷贝”表头”? slice 的 len/cap/底层数组指针 三元组如何工作?goroutine 为什么便宜?逃逸分析如何决定栈还是堆?

阅读顺序(建议)

全教程贯穿的几个”底层视角”

  1. Go 里赋值和传参永远是”值拷贝”——理解这一点后,slice/map/channel 的”看起来像引用”的行为都能解释:它们拷贝的是一个很小的”表头结构”,表头里含有指向底层数据的指针。
  2. 指针是显式且安全的:有 &、*,但没有指针算术;编译器通过逃逸分析决定变量放栈还是堆,GC 负责回收。
  3. 并发是语言内建的:go f() 启动一个由运行时调度到 OS 线程上的 goroutine,channel 是带锁的类型安全队列——不是库,是语言原语。
