UML 类图基础 · UML Class Diagrams

面试白板的类图不求工具标准，但要求：类名、关键字段/方法、关系箭头 画清楚，能支撑你讲设计。

一、为什么要画类图？

LLD 面试里，类图是「设计说明书」：

  1. 强迫你先想实体与关系，再写代码
  2. 方便和面试官对齐：缺不缺类、职责对不对
  3. 暴露耦合：箭头太多、环状依赖 = 设计味道

口诀：名词成框，关系成线，变化画成接口。

二、类的画法

标准三格：类名 / 属性 / 方法。白板可简化成两格或只写关键方法。

┌─────────────────────┐
│      BankAccount    │  ← 类名（抽象类可斜体或标 <<abstract>>）
├─────────────────────┤
│ - balance: float    │  ← 属性（- private  # protected  + public）
├─────────────────────┤
│ + deposit(amount)   │  ← 方法
│ + withdraw(amount)  │
│ + get_balance()     │
└─────────────────────┘

接口 / 协议

┌─────────────────────┐
│   <<interface>>     │
│   PaymentGateway    │
├─────────────────────┤
│ + charge(amount)    │
└─────────────────────┘

Python 面试可写 ABC / Protocol，语义都是「契约」。

可见性 符号 面试建议
public + 对外 API
private - 内部状态
protected # 给子类用（少画）

三、关系一览

关系 含义 箭头（ASCII） 代码直觉
关联 Association 互相知道 A ----> B 持有引用
聚合 Aggregation 整体-部分，部分可独立 A <>--- B list 里有 B，B 可别处活
组合 Composition 整体-部分，生命周期绑定 A ◆--- B A 销毁 B 也没意义
继承 Inheritance is-a A ---> B（空心三角指向父） class A(B)
实现 Realization 实现接口 A ..|> I class A(I)
依赖 Dependency 临时用一下 A - - > B 方法参数/局部变量

聚合 vs 组合（常考）

大学 ◇—— 学生     学生离开大学仍存在 → 聚合
房子 ◆—— 房间     房子拆了房间没了   → 组合
订单 ◆—— 订单行   订单删除行项无意义 → 组合

口诀：生命周期绑死用组合（实心菱形）；部分可独立用聚合（空心菱形）。

五、面试白板怎么画（实操）

推荐顺序（5–8 分钟）

1. 列出名词 → 画方框（只写类名）
2. 标「谁拥有谁」→ 组合/聚合
3. 找出变化点 → 画接口 + 实现类
4. 补关键方法（park / checkout / search…）
5. 口头走一遍主流程，改漏的箭头

时间不够时的简化规则

保留 可省略
核心实体 + 接口 getter/setter、DTO
组合/继承箭头 每个字段类型
3–5 个关键方法 异常类全家桶
策略/工厂框 工具函数

口述模板

「ParkingLot 组合多个 Floor；Floor 组合 Spot。计费用 FeeStrategy，Lot 依赖接口不依赖具体公式。车辆用 Vehicle 抽象，Car/Truck 实现不同占位规则。」

六、常见画错

错误 问题 改法
所有关系都画成实线箭头 分不清拥有/依赖 组合用菱形，依赖用虚线
为每个字段画一个类 图爆炸 值对象可写在属性里
God 类居中连一切 SRP 失败 拆服务 / 仓储 / 策略
继承表示「用到」 LSP 隐患 改为关联或依赖
接口与实现画反 三角方向错 空心三角指向父/接口

七、从需求到类图小练习

需求：「图书馆可借还书；会员有借阅上限；支持按书名搜索。」

候选名词

Library, Member, Book, BookItem(馆藏副本), Loan, Catalog

草图

Library ◆── Catalog
Library ◆── members: Member
Catalog ----> Book（按 ISBN,索引持有引用,用关联而非聚合/组合)
Book ◇── BookItem（可借副本）
Member ◇── Loan
Loan ----> BookItem
Loan ----> Member

方法提示：borrow(member, item)、return(item)、search(title)。

完整案例见：图书管理系统。

八、与代码的对应

图上 Python
类框 class Foo:
<<interface>> ABC / Protocol
实心菱形 在 __init__ 里创建并持有
空心菱形 外部传入或注册
依赖虚线 参数类型、局部 new
空心三角 class Sub(Base)
