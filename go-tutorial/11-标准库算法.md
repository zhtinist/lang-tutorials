# 标准库中的算法与数据结构

C++ 程序员熟悉 STL:`std::sort`、`std::vector`、`std::map`、`std::priority_queue`、迭代器与算法解耦。Go 标准库的哲学不同:早期没有泛型,容器要么内建(slice/map),要么靠**接口 + 手写实现**;直到 Go 1.18/1.21 才补上泛型工具函数。本章过一遍排序、泛型 `slices`/`maps`、`container/*` 三剑客,以及字符串/数值工具,并解释这段"从无泛型到有泛型"的历史转变。

本章偏工具速查,风格和前几章不同:**每一节都是先给一段能直接抄的用法示例,再补充复杂度、内部算法或和 C++ 对应设施的差异**——不需要专门分「怎么用」和「底层」两段读,跟着代码顺序往下看即可。

## 一、sort 包:接口式排序

### sort.Slice —— 最常用

```go
people := []Person{{"Bob", 31}, {"Alice", 25}, {"Eve", 40}}
sort.Slice(people, func(i, j int) bool {   // less(i,j): i 是否应排在 j 前
    return people[i].Age < people[j].Age
})
```

`sort.Slice` 用反射拿到 slice 并做元素交换,less 闭包定义顺序。不稳定;要稳定用 `sort.SliceStable`。

### sort.Sort —— 接口式(泛型前的经典做法)

排任意集合,只要它实现 `sort.Interface` 三个方法。这非常"Go 1.0 风格"——**用接口而非模板做多态**:

```go
type sort.Interface interface {
    Len() int
    Less(i, j int) bool
    Swap(i, j int)
}

type ByAge []Person
func (a ByAge) Len() int           { return len(a) }
func (a ByAge) Less(i, j int) bool { return a[i].Age < a[j].Age }
func (a ByAge) Swap(i, j int)      { a[i], a[j] = a[j], a[i] }

sort.Sort(ByAge(people))
```

便捷函数:`sort.Ints`、`sort.Strings`、`sort.Float64s`。

### 内部算法:pattern-defeating quicksort (pdqsort)

Go 1.19 起,`sort` 内部用 **pdqsort**(模式击败快排),而不是朴素快排。它是一种混合排序:

- **主体是快速排序**,但用了更聪明的 pivot 选择(中位数取样),尽量避免退化。
- **小数组(约 ≤12 个元素)切换到插入排序**——小规模下常数更优、缓存友好。
- **遇到过多"坏划分"时切换到堆排序兜底**——pdqsort 维护一个 `limit` 计数器(初值约 `log₂n`),每次碰到极不平衡的 pivot 划分就递减,归零时改用堆排序(注意这**不是**经典 introsort"按递归深度 2·log n"的判据)。由此防止对手构造的输入把快排拖到 O(n²),保证**最坏 O(n log n)**。
- 能识别已排序、有大量重复元素等"模式",走快速路径。

这与 C++ STL 的 `std::sort`(introsort:快排 + 堆排 + 插入排序)思路一脉相承,pdqsort 是更新的改进版。

| 函数 | 稳定 | 平均 | 最坏 |
|------|------|------|------|
| `sort.Sort`/`sort.Slice` | 否(pdqsort) | O(n log n) | O(n log n) |
| `sort.Stable`/`sort.SliceStable` | 是(原地归并) | O(n log n) 比较 | O(n log² n)(交换更多,原地不需 O(n) 额外空间) |

### sort.Search —— 通用二分查找

```go
// 在已升序的 a 中找第一个 >= target 的下标(相当于 lower_bound)
i := sort.Search(len(a), func(i int) bool { return a[i] >= target })
if i < len(a) && a[i] == target { /* 找到 */ }
```

`sort.Search` 找的是"使谓词从 false 变 true 的第一个位置",O(log n)。谓词必须单调。比手写二分不易错。

## 二、泛型 slices / maps 包(Go 1.21)

Go 1.21 把泛型工具函数正式纳入标准库 `slices` 和 `maps`,类型安全、无需反射、无需写 `sort.Interface`。这是对 C++ `<algorithm>` 迟到的回应。

### slices 包

```go
import "slices"

s := []int{3, 1, 2}
slices.Sort(s)                     // 升序, 内部 pdqsort, 无反射(比 sort.Slice 快)
slices.SortFunc(people, func(a, b Person) int {
    return cmp.Compare(a.Age, b.Age) // 返回 <0/0/>0(注意不是 bool!)
})
i, found := slices.BinarySearch(s, 2)  // 二分, 返回下标和是否命中
ok := slices.Contains(s, 2)
idx := slices.Index(s, 2)              // 找不到返回 -1
mx := slices.Max(s)                    // 也有 Min
eq := slices.Equal(s, []int{1, 2, 3})
s = slices.Insert(s, 1, 99)            // 在下标1插入
s = slices.Delete(s, 0, 1)            // 删除 [0,1)
r := slices.Clone(s)                   // 浅拷贝(见第07章别名问题)
s = slices.Compact(s)                  // 去除相邻重复
```

注意 `SortFunc` 的比较函数返回 **`int`(三态:负/零/正)**,不是 `sort.Slice` 那种 `bool`——和 C 的 `qsort` 比较器一致,能表达相等。配合 `cmp` 包的 `cmp.Compare`/`cmp.Or` 用。

可运行示例:`slices.Sort`、`slices.BinarySearch`、以及用 `cmp.Compare` 做多级排序的 `slices.SortFunc`:

```go
package main

import (
	"cmp"
	"fmt"
	"slices"
)

type Person struct {
	Name string
	Age  int
}

func main() {
	nums := []int{3, 1, 4, 1, 5, 9, 2, 6}
	slices.Sort(nums) // 升序, 无反射
	fmt.Println(nums)
	// [1 1 2 3 4 5 6 9]

	i, found := slices.BinarySearch(nums, 5) // 要求已排序
	fmt.Println(i, found)
	// 5 true

	people := []Person{{"Bob", 31}, {"Alice", 25}, {"Eve", 25}}
	slices.SortFunc(people, func(a, b Person) int {
		if c := cmp.Compare(a.Age, b.Age); c != 0 {
			return c // 先按年龄
		}
		return cmp.Compare(a.Name, b.Name) // 年龄相同再按名字
	})
	fmt.Println(people)
	// [{Alice 25} {Eve 25} {Bob 31}]
}
```

### maps 包

map 本身无序、无内建"取所有键"的操作,`maps` 包补上:

```go
import "maps"

m := map[string]int{"a": 1, "b": 2}
ks := maps.Keys(m)      // Go1.23: 返回迭代器 iter.Seq[K]
vs := maps.Values(m)
m2 := maps.Clone(m)     // 浅拷贝
maps.Copy(dst, src)     // 把 src 拷进 dst
eq := maps.Equal(m, m2)
```

> 版本提示:`maps.Keys`/`Values` 的签名在 Go 1.23 随 range-over-func 迭代器改成返回 `iter.Seq`。要拿到排好序的键,常见写法:`ks := slices.Sorted(maps.Keys(m))`(1.23+)。以你的 Go 版本文档为准。

可运行示例:`maps.Keys` 返回迭代器,`slices.Sorted` 收集并排序,得到确定的遍历顺序:

```go
package main

import (
	"fmt"
	"maps"
	"slices"
)

func main() {
	m := map[string]int{"banana": 3, "apple": 5, "cherry": 1}

	// maps.Keys 返回迭代器(Go 1.23+); slices.Sorted 收集并排序
	for _, k := range slices.Sorted(maps.Keys(m)) {
		fmt.Printf("%s=%d\n", k, m[k])
	}
	// apple=5
	// banana=3
	// cherry=1

	total := 0
	for v := range maps.Values(m) {
		total += v
	}
	fmt.Println("total:", total)
	// total: 9
}
```

## 三、container/* 三剑客

标准库唯三的"容器"包,都是**接口式**、非泛型的老设计(至今未泛型化)。

### container/heap —— 二叉堆(优先队列)

不是拿来即用的类,而是一组作用于 `heap.Interface` 的**算法**。你得先实现 `sort.Interface` 三方法**再加** `Push`/`Pop` 两个,共 5 个方法,底层用一个 slice 存堆:

```go
type IntHeap []int
func (h IntHeap) Len() int            { return len(h) }
func (h IntHeap) Less(i, j int) bool  { return h[i] < h[j] } // 小顶堆
func (h IntHeap) Swap(i, j int)       { h[i], h[j] = h[j], h[i] }
func (h *IntHeap) Push(x any)         { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() any {
    old := *h; n := len(old)
    x := old[n-1]; *h = old[:n-1]
    return x
}

h := &IntHeap{5, 2, 8}
heap.Init(h)              // O(n) 建堆
heap.Push(h, 1)          // O(log n)
min := heap.Pop(h)       // O(log n), 弹出最小
```

对照 C++ `std::priority_queue`:功能相同但 Go 要你手写 5 个方法(泛型前的代价),`Push`/`Pop` 用 `any` 装箱。这是"Go 早期无泛型"最直白的痛点样本。

| 操作 | 复杂度 |
|------|--------|
| Init | O(n) |
| Push / Pop | O(log n) |
| 取堆顶(h[0]) | O(1) |

可运行示例:一个完整的小顶堆,`Init`/`Push`/`Pop` 后依次弹出即得升序:

```go
package main

import (
	"container/heap"
	"fmt"
)

// IntHeap 是一个小顶堆, 实现 heap.Interface 的 5 个方法
type IntHeap []int

func (h IntHeap) Len() int           { return len(h) }
func (h IntHeap) Less(i, j int) bool { return h[i] < h[j] } // < 即小顶堆
func (h IntHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *IntHeap) Push(x any)        { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() any {
	old := *h
	n := len(old)
	x := old[n-1]
	*h = old[:n-1]
	return x
}

func main() {
	h := &IntHeap{5, 2, 8, 1}
	heap.Init(h)                 // O(n) 建堆
	heap.Push(h, 3)              // O(log n)
	fmt.Println("top:", (*h)[0]) // 堆顶始终最小

	var out []int
	for h.Len() > 0 {
		out = append(out, heap.Pop(h).(int)) // 依次弹出 → 升序
	}
	fmt.Println("popped:", out)
	// top: 1
	// popped: [1 2 3 5 8]
}
```

### container/list —— 双向链表

```go
l := list.New()
e := l.PushBack(1)       // 尾插, 返回 *list.Element
l.PushFront(0)           // 头插
l.InsertAfter(2, e)
for e := l.Front(); e != nil; e = e.Next() {
    fmt.Println(e.Value) // Value 是 any, 用时断言
}
```

元素是节点(含 prev/next 指针 + `any` 值),增删 O(1)。但实际中**大多数场景 slice 更快**(连续内存、缓存友好),链表只在频繁中间增删且持有节点引用时才划算——和 C++ `std::list` vs `std::vector` 的取舍一样。

### container/ring —— 环形链表

固定循环缓冲的双向环,`r.Next()`/`r.Prev()` 转圈,`r.Move(n)`。用得少,做定长滑动窗口偶尔用。

## 四、字符串与字节:strings / bytes / strconv

### strings

`string` 是只读字节序列([第04章](04-数据类型.md)),`strings` 包提供操作,**返回新字符串(不可变)**:

```go
strings.Contains(s, "go")     strings.HasPrefix / HasSuffix
strings.Split(s, ",")         strings.Join(parts, "-")
strings.ToUpper / ToLower     strings.TrimSpace / Trim
strings.ReplaceAll(s, "a", "b")   strings.Fields(s) // 按空白切
strings.Index(s, "x")             strings.Repeat("ab", 3)
```

大量拼接用 `strings.Builder`(内部可增长 `[]byte`,避免每次拼接都分配新字符串——类似 C++ 用 `string::reserve` + `+=`):

```go
var b strings.Builder
for i := 0; i < 100; i++ { b.WriteString("x") }
result := b.String()
```

### bytes

和 `strings` 几乎镜像的 API,但作用于 `[]byte`(可变)。`bytes.Buffer` 是可读写缓冲,常用作 `io.Reader`/`io.Writer`。处理二进制、拼装网络协议、避免 `string`↔`[]byte` 拷贝时用它。

### strconv

字符串与数值/布尔互转(比 `fmt` 快,无反射):

```go
n, err := strconv.Atoi("42")            // string → int
s := strconv.Itoa(42)                    // int → string
f, err := strconv.ParseFloat("3.14", 64)
b, err := strconv.ParseBool("true")
s = strconv.FormatInt(255, 16)           // "ff"
s = strconv.Quote(`he said "hi"`)        // 加转义引号
```

## 五、math / math/rand

```go
math.MaxInt64, math.Pi, math.Inf(1), math.NaN()
math.Sqrt, math.Pow, math.Floor, math.Abs, math.Max/Min
math.MaxInt, math.MaxUint32 ...          // 各类型边界常量
```

随机数:

```go
import "math/rand/v2"                     // Go1.22+ 新版, 推荐
n := rand.IntN(100)                       // [0,100)
f := rand.Float64()                       // [0.0,1.0)
rand.Shuffle(len(s), func(i, j int) { s[i], s[j] = s[j], s[i] })
```

> `math/rand` **不是密码学安全**的(可预测)。需要安全随机(令牌、密钥、盐)用 `crypto/rand`。**Go 1.20 起**顶层 `math/rand` 已默认随机播种、`rand.Seed` 被弃用,不必再手动 Seed(仅 1.20 之前需要);`math/rand/v2`(1.22)是更干净的新 API。

## 六、复杂度速查

| 结构/操作 | 复杂度 | 备注 |
|-----------|--------|------|
| slice 索引 `s[i]` | O(1) | 带边界检查 |
| slice `append`(cap 够) | O(1) 摊销 | 扩容时 O(n) 拷贝 |
| map 查/增/删 | O(1) 平均 | 最坏退化;顺序随机 |
| `sort.Slice`/`slices.Sort` | O(n log n) | pdqsort,最坏也 O(n log n) |
| `sort.Search`/`slices.BinarySearch` | O(log n) | 要求已排序 |
| heap Push/Pop | O(log n) | 堆顶 O(1) |
| list 增删(持有节点) | O(1) | 遍历 O(n) |

## 七、历史:Go 为什么曾经没有泛型容器

早期 Go(1.0~1.17)**没有泛型**,这直接塑造了标准库的形态:

- 容器只有内建的 slice/map/chan(编译器特殊支持,可对任意元素类型工作)。
- 通用算法只能靠**接口 + 反射**:`sort.Sort` 要你实现 `sort.Interface`;`container/heap`/`list` 用 `any`(旧称 `interface{}`)存元素,取出要类型断言,有装箱开销、无编译期类型检查。
- 想要"类型安全的栈/队列/集合",要么复制粘贴具体类型版本,要么用 `interface{}` 牺牲类型安全,要么代码生成(`go generate`)。这是社区多年最大的抱怨。

设计者当年的理由:泛型会增加语言复杂度,担心重蹈 C++ 模板的复杂与编译膨胀,宁可先不做也不做错。经过多轮提案,**Go 1.18(2022)引入类型参数**([第08章泛型机制](08-面向对象与类型系统.md)),**Go 1.21(2023)**把泛型工具沉淀为标准库 `slices`/`maps`/`cmp`,`min`/`max`/`clear` 成为内建函数。至此 Go 才有了类型安全、无反射的通用算法。

转变的实际影响:

- 新代码优先用 `slices.Sort` 而非 `sort.Slice`(前者无反射、更快、类型安全)。
- `container/heap`/`list` 这类老接口式容器**没有**被泛型化(向后兼容),但社区已有大量泛型容器库可选。
- 泛型让"写一个类型安全的 `Map`/`Filter`/`Reduce`"成为可能,但 Go 团队仍克制,标准库只加最基础的,不追求 C++/Rust 那样庞大的算法族。

一句话:Go 从"用接口做多态、容器靠内建 + 反射"演进到"泛型 + 内建混合",但整体仍比 STL 精简得多——**够用、简单优先**是它一贯的取舍。
