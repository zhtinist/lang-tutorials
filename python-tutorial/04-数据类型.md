# 数据类型

本章介绍 Python 的常用内置数据类型。**每种类型都先讲「怎么写、怎么用」——你不需要懂底层也能马上上手;再进入「底层与 C 对比」——看它在 CPython 里到底长什么样、和你熟悉的 C 定长 `int`、IEEE-754 `double`、`char*` 差在哪。** 只想会用的话,读每节的前半段即可;想搞懂性能与内存,再往下看后半段。

贯穿本章的一句话:**在 C 里类型决定了变量占几个字节;在 Python 里类型是对象堆里的一块结构体,变量只是指向它的指针。**

---

## 4.1 先会用:内置类型与字面量速览

写 Python 之前,先认识怎么「写出」各种值。下面这些字面量不需要任何 import:

```python
n = 42              # int   整数,任意大小
f = 3.14            # float 浮点数
b = True            # bool  布尔(True / False)
z = 3 + 4j          # complex 复数(虚部后缀 j)
s = "café😀"        # str   字符串(Unicode 文本)
raw = b"hello"      # bytes 字节序列(前缀 b)
nothing = None      # 空值
```

常见的即时操作:

```python
print(7 // 2)       # 3    —— 整除(向下取整)
print(7 % 2)        # 1    —— 取余
print(2 ** 10)      # 1024 —— 幂运算
print("ab" * 3)     # ababab —— 字符串重复
print(len("café"))  # 4    —— 长度(数的是字符,不是字节)
print(type(42))     # <class 'int'> —— 查类型
```

用类型名当函数做转换:

```python
int("42")           # 42
float("3.14")       # 3.14
str(255)            # "255"
```

这些够你开始写代码了。下面逐个类型展开——**先用法,后底层**。

## 4.2 开始深入:所有值都是堆对象

从这一节起进入「底层」。要理解后面每种类型的内存开销,先记住一个事实:**CPython 里每个值都是堆上的一个对象**,开头都有一段统一的「对象头」。

```c
// CPython 源码 Include/object.h(简化)
typedef struct _object {
    Py_ssize_t    ob_refcnt;    // 引用计数
    PyTypeObject *ob_type;      // 指向类型对象的指针(相当于 C++ 的 vptr)
} PyObject;

typedef struct {
    PyObject ob_base;
    Py_ssize_t ob_size;         // 变长对象的"元素个数"
} PyVarObject;
```

`ob_type` 就是 C++ 里虚表指针(vptr)的类比:它指向一个 `PyTypeObject`,里面存着「这个类型怎么加法、怎么打印、怎么取长度」等一堆函数指针(称为 tp_slots)。区别在于 C++ 的 vtable 在编译期定死,而 Python 的类型对象是运行期可修改的普通对象。

在 64 位 CPython 上,这个头是 16 字节(`PyObject`)或 24 字节(`PyVarObject`)。**任何 Python 对象都至少这么大**——这也是为什么 Python 的 `int` 动辄二三十字节,而 C 的 `int` 只有 4 字节。

```python
import sys
print(sys.getsizeof(0))      # 28(64 位 CPython 3.12+;更早版本是 24)
print(sys.getsizeof(1.0))    # 24
```

记住这个「对象头」,下面每种类型的「底层」小节都会用到它。

## 4.3 `int`:任意精度整数

### 怎么用

Python 的整数**没有大小上限**,想多大就多大,不会像 C 那样溢出:

```python
x = 2 ** 100
print(x)            # 1267650600228229401496703205376,不溢出
print(x * x)        # 继续放大也没问题

print(10 // 3)      # 3   —— 整除
print(10 % 3)       # 1   —— 取余
print(-7 // 2)      # -4  —— 注意:向下取整(不是向零),和 C 不同!
print(int("ff", 16))# 255 —— 按 16 进制解析
```

> ⚠️ 一个和 C 不同的坑:Python 的 `//` 和 `%` 是**向下取整**(floor),`-7 // 2 == -4`;C 的整数除法是**向零截断**,`-7 / 2 == -3`。

### 底层与 C 对比

Python 的 `int` 是**任意精度**的:它把整数存成一个「digit 数组」,像小学竖式一样按需扩展位数——本质上就是 C 里手写的 bignum(GMP 那类库干的事),只不过内置了。

```c
// 简化后的概念结构
typedef struct {
    PyObject_VAR_HEAD          // ob_refcnt, ob_type, ob_size
    digit ob_digit[1];         // 变长数组,每个 digit 存 30 位
} PyLongObject;
```

关键点:

- 每个 `digit` 是一个 32 位无符号整数(`uint32_t`),但**只用其中 30 位**(`PYLONG_BITS_IN_DIGIT == 30`)。留出的位方便进位、借位运算不溢出 C 的 32 位。
- `ob_size` 的**绝对值**是 digit 个数,**符号**表示整数正负;`ob_size == 0` 表示数值 0。这是「符号-数值」表示法,不是补码。

> 注:CPython 3.12 起内部改用 `ob_size` + 一个标志位打包的 `lv_tag`(属实现细节),但「30-bit digit 数组 + 符号」这个模型不变。

内存布局示意,`2**60`(需要 3 个 30-bit digit):

```
值 = digit[0] * 2^0 + digit[1] * 2^30 + digit[2] * 2^60

+-------------------+
| ob_refcnt         |  \
| ob_type  --> int  |   > 对象头 (PyVarObject)
| ob_size = 3       |  /   (正号 + 3 个 digit)
+-------------------+
| ob_digit[0] (30b) |
| ob_digit[1] (30b) |
| ob_digit[2] (30b) |
+-------------------+
```

和 C 对照:

| | C `int32_t` | Python `int` |
|---|---|---|
| 大小 | 固定 4 字节 | 变长,随数值增大 |
| 溢出 | 回绕(UB/实现定义) | 不溢出,自动扩容 |
| 负数表示 | 二进制补码 | 符号位 + 数值(digit 数组) |
| 位置 | 可在栈上/寄存器 | 永远在堆上,变量持指针 |
| 运算 | 单条 CPU 指令 | 函数调用,逐 digit 处理 |

用 `sys.getsizeof` 观察增长:

```python
import sys
print(sys.getsizeof(0))          # 28(空 digit 情况,依版本)
print(sys.getsizeof(2**29))      # 28  —— 1 个 digit
print(sys.getsizeof(2**30))      # 32  —— 需要 2 个 digit
print(sys.getsizeof(2**60))      # 36  —— 3 个 digit
```

每多一个 30-bit digit,大约多 4 字节,和上面的结构完全吻合。

**小整数缓存**:CPython 启动时预先创建了 `[-5, 256]` 范围内的所有整数对象,永久驻留。所以:

```python
a = 100
b = 100
print(a is b)                       # True —— 同一个缓存对象
print(int("1000") is int("1000"))   # False:运行时各自新建,落在缓存范围外
```

**永远不要用 `is` 比较数值**,用 `==`。

## 4.4 `bool`:`True` / `False`

### 怎么用

```python
flag = True
print(not flag)         # False
print(True and False)   # False
print(3 > 2)            # True —— 比较运算返回 bool
```

### 底层与 C 对比

Python 里 `bool` **继承自 `int`**,`True` 和 `False` 就是值为 1 和 0 的两个单例整数对象:

```python
print(isinstance(True, int))   # True
print(True + True)             # 2 —— 当成整数参与运算
print(True == 1)               # True
```

这和 C 的 `bool`(`_Bool`,通常 1 字节栈上值)完全不同:Python 的 bool 是堆上一个继承自 int 的对象。一个实用后果——可以直接对布尔求和来计数:

```python
data = [1, -2, 3, -4, 5]
positives = sum(x > 0 for x in data)   # 把 True/False 当 1/0 求和 = 3
```

## 4.5 `float`:双精度浮点

### 怎么用

```python
f = 3.14
print(f + 1)            # 4.140000000000001
print(2.0 ** 0.5)       # 1.4142135623730951 —— 开方
print(round(3.14159, 2))# 3.14

print(0.1 + 0.2)        # 0.30000000000000004 —— 浮点误差!
print(0.1 + 0.2 == 0.3) # False —— 别用 == 比浮点
```

要精确的十进制(金融)用 `decimal.Decimal`,精确分数用 `fractions.Fraction`。

### 底层与 C 对比

Python 的 `float` 就是一个 IEEE-754 双精度浮点,**装箱**(boxing)在对象里:

```c
typedef struct {
    PyObject_HEAD
    double ob_fval;      // 一个货真价实的 C double
} PyFloatObject;
```

所以一个 float 对象 = 16 字节对象头 + 8 字节 double = 24 字节:

```python
import sys
print(sys.getsizeof(3.14))   # 24
```

C 的 `double x = 3.14;` 是栈上 8 字节裸值,运算直接走 FPU 指令。Python 里每个 float 是堆对象,`a + b` 要解引用两个指针取出 `ob_fval`、相加、再**分配一个新 float 对象**装结果。这就是数值密集计算下纯 Python 慢、内存开销大的根源,也是 NumPy 存在的理由——NumPy 把一亿个 double 打包进一块连续 C 数组,而不是一亿个 24 字节对象([见「常用第三方库」](12-常用第三方库.md))。

浮点的所有「坑」和 C 完全一样,因为底层就是同一个 double:上面 `0.1 + 0.2 != 0.3` 就是明证。特殊值也一致:`float('inf')`、`float('nan')`,且 `nan != nan`。

## 4.6 `complex`:复数

### 怎么用

虚部字面量后缀是 `j`,内置类型,不需要 import:

```python
z = 3 + 4j
print(z.real, z.imag)   # 3.0 4.0
print(abs(z))           # 5.0 —— 模长
```

要做复数的三角/指数运算,用标准库 `cmath`(而不是 `math`):

```python
import cmath
z = 1 + 1j
print(cmath.phase(z))   # 0.7853981633974483 —— 辐角(π/4)
print((-1 + 0j) ** 0.5) # (6.12e-17+1j) —— √-1 ≈ i(实部是浮点噪声)
```

### 底层与 C 对比

`complex` 就是一个装了两个 C double(实部、虚部)的对象:

```c
typedef struct {
    PyObject_HEAD
    double real;
    double imag;
} PyComplexObject;
```

和 `float` 同理,只是多一个 double。注意 `(-1) ** 0.5` 对 `float` 会返回 `nan`,但对 `complex` 走复数开方得到约等于 `i`。

## 4.7 `str`:Unicode 文本

### 怎么用

字符串是不可变的 Unicode 文本,可索引、可切片,但不能原地改:

```python
s = "café😀"
print(s[0])             # c   —— 索引取单个字符
print(s[1:4])           # afé —— 切片
print(s.upper())        # CAFÉ😀
print("a" in s)         # False
print(len(s))           # 5   —— 数的是「字符(码点)」,不是字节!

# s[0] = "C"            # TypeError:字符串不可变
s2 = s.replace("c", "C")# 任何「修改」都产生新字符串
```

和外部(文件、网络)打交道时,`str`(文本)与 `bytes`(字节)必须显式编解码:

```python
s = "café"
data = s.encode("utf-8")     # str -> bytes: b'caf\xc3\xa9'
back = data.decode("utf-8")  # bytes -> str: café
```

### 底层与 C 对比

C 的 `char*` 是一串以 `\0` 结尾的字节;Python 的 `str` 是**不可变的 Unicode 码点序列**,且 CPython 会根据内容选最省的内部编码。

**PEP 393**:从 Python 3.3 起,CPython 看字符串里**最大的码点**,选三种紧凑表示之一:

| 内部种类 | 每字符字节 | 可表示范围 | 例子 |
|---|---|---|---|
| Latin-1 | 1 | U+0000–U+00FF | `"hello"`, `"café"` |
| UCS-2 | 2 | U+0000–U+FFFF | `"你好"` |
| UCS-4 | 4 | U+0000–U+10FFFF | `"😀"` |

只要字符串里出现一个高码点字符,**整个字符串**升到更宽的表示:

```python
import sys
print(sys.getsizeof("a"))       # 42 —— Latin-1,每字符 1 字节
print(sys.getsizeof("你"))      # 60 —— UCS-2,每字符 2 字节
print(sys.getsizeof("😀"))      # 64 —— UCS-4,每字符 4 字节
```

与 C 对照:

| | C `char*` | Python `str` |
|---|---|---|
| 单元 | 字节 | Unicode 码点 |
| 长度 | 靠 `\0` 结尾,O(n) 求长 | 显式存 length,O(1) |
| `len("😀")` | 按字节数(4,UTF-8) | 1(一个码点) |
| 可变性 | 可改 | 不可变 |

`len()` 数的是**码点**不是字节,这点常让写惯 C 的人踩坑:

```python
s = "café😀"
print(len(s))            # 5 个码点
print(len(s.encode()))   # 9 个 UTF-8 字节:c/a/f 各 1 + é(2) + 😀(4)
```

> **别把两件事搞混**:`.encode()` 得到的是 **UTF-8 变长编码**(每字符按码点占 1–4 字节),和它在内存里怎么存**无关**;而 PEP 393 说的"整串升到 4 字节/字符"指的是 `str` **对象在内存里的定宽缓冲区**。内存里用**定宽**存(便于 `s[i]` O(1) 索引),`.encode()` 序列化成 UTF-8 用**变长**存(便于存储/传输)。

因为不可变,字符串可以缓存哈希值,成为高效的 dict 键。CPython 还对看起来像标识符的短字符串字面量做**驻留**(interning),让 `is` 快速比较、省内存——但比较内容仍然只用 `==`。

## 4.8 `bytes` / `bytearray`:才是「C 的 char 数组」

### 怎么用

如果你想要 C 里那个「一串裸字节」,对应的是 `bytes`(不可变)和 `bytearray`(可变),不是 `str`:

```python
b = b"hello"          # bytes 字面量,前缀 b
print(b[0])           # 104 —— 取出的是 int(字节值),不是字符

ba = bytearray(b"hi") # 可变版本
ba.append(33)         # 追加 '!' (0x21)
ba[0] = 72            # 原地修改 -> 'H'
print(ba)             # bytearray(b'Hi!')
```

解析二进制协议时,把字节按大小端拼成整数——C 里做移位和 `|`,Python 直接用 `int.from_bytes` / `int.to_bytes`:

```python
data = b'\x00\x00\x01\x00'
print(int.from_bytes(data, "big"))     # 256   —— 大端
print(int.from_bytes(data, "little"))  # 65536 —— 小端
print((1000).to_bytes(4, "big").hex()) # 000003e8
```

### 底层与 C 对比

- `bytes`:不可变字节序列,内部就是一段连续的 `unsigned char` 缓冲 + 长度。最接近 C 的 `const char*` + 长度。
- `bytearray`:可变版本,内部像 C++ 的 `std::vector<uint8_t>`——有容量、可原地增删。

想要 C 里那种「拿一个指针指进缓冲区、零拷贝读写」,对应的是 `memoryview`:它不复制底层字节,只是开在已有 buffer 上的视图,切片也零拷贝(类比 C++ 的 `std::span` / `string_view`):

```python
ba = bytearray(b"hello world")
mv = memoryview(ba)          # 零拷贝视图
chunk = mv[6:11]             # 切片仍是视图,指向原缓冲(没有新分配)
chunk[0] = ord('W')          # 通过视图原地改
print(ba)                    # bytearray(b'hello World') —— 原对象被改了
```

处理大块二进制(网络包、图像)时,这能避免 `data[a:b]` 那种「切一次就复制一份」的开销。C 里 `char*` 既是文本又是字节、编码全靠约定;Python 强制你在边界显式编解码,`str` 处理文本、`bytes` 处理二进制。

## 4.9 单例:`None`、`NotImplemented`、`Ellipsis`

这三个是全程唯一的单例对象,用法本身就很简单。

`None` 表示「无值/空」,类比 C 的 `NULL`,但它是真正的对象(类型 `NoneType`),不是 0 指针:

```python
x = None
print(x is None)        # 判空的标准写法,永远用 is
```

`None` 全程只有一个实例,用 `is None` 既快又不会被自定义 `__eq__` 干扰。函数没有 `return` 时默认返回 `None`。

`NotImplemented` 用在运算符重载里,某类型不知道怎么和对方运算时返回它(不是抛异常),让 Python 去尝试对方的反射方法:

```python
class Vec:
    def __init__(self, x): self.x = x
    def __add__(self, other):
        if not isinstance(other, Vec):
            return NotImplemented    # 交给对方处理
        return Vec(self.x + other.x)
```

`Ellipsis`(`...`)字面量就是三个点,常用作占位符,或被 NumPy 用作多维切片(`arr[..., 0]`):

```python
def todo(): ...          # 合法的函数体占位(等价于 pass)
print(... is Ellipsis)   # True
```

## 4.10 数值与字符串转换

显式构造/转换用类型名当函数(其实是调用类型对象):

```python
int("42")           # 42        —— str -> int
int("ff", 16)       # 255       —— 指定进制
int(3.9)            # 3         —— float 截断(向零取整)
float("3.14")       # 3.14
str(255)            # "255"
bin(255)            # "0b11111111"
hex(255)            # "0xff"
```

格式化(f-string,3.6+)是最常用的数值转字符串方式:

```python
n = 255
print(f"{n:d} {n:b} {n:x} {n:08b}")   # 255 11111111 ff 11111111
pi = 3.14159
print(f"{pi:.2f}")                    # 3.14
print(f"{1234567:,}")                 # 1,234,567 —— 千位分隔
```

转换失败会抛 `ValueError`,而不是像 C 的 `atoi` 那样静默返回 0:

```python
try:
    int("abc")
except ValueError as e:
    print("转换失败:", e)
```

这种「出错就抛异常」的取向,是 Python 相对 C 返回错误码的一大风格差异,详见 [见「错误处理与资源管理」](09-错误处理与资源管理.md)。

## 小结

- **用法层面**:字面量直接写(`42` / `3.14` / `"文本"` / `b"字节"`);`//` 是整除且向下取整、`**` 是幂;`len` 数的是码点;字符串不可变;转换用类型名调用、失败抛 `ValueError`。
- **底层层面**:所有值都是堆对象,共享 `PyObject` 头(引用计数 + 类型指针 ≈ vptr)。
  - `int` 是变长 bignum(30-bit digit 数组 + 符号),不溢出;`[-5,256]` 有缓存单例。
  - `bool` 是 `int` 子类;`float`/`complex` 是装箱的 C double(一个/两个)。
  - `str` 是不可变 Unicode,按最大码点选 1/2/4 字节紧凑存储(PEP 393);`bytes`/`bytearray` 才对应 C 的字节数组,`memoryview` 对应零拷贝视图。
