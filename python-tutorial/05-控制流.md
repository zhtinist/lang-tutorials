# 控制流

控制流的关键字看起来和 C 差不多:`if`/`while`/`for`。但底层机制差得远。C 的 `for (int i=0; i<n; i++)` 是编译期生成的整数递增 + 比较跳转;Python 的 `for` 是**运行时的方法调用协议**,每一次循环都在向一个对象发消息(`__next__`)。这一章的重点就是把这些"看似熟悉"的结构拆开,看底层到底在做什么。

## 真值测试:不是只有 0 和 1

C 里 `if (x)` 就是判断 `x != 0`(整数、指针都一样)。Python 里 `if x:` 触发的是一套**协议**,任何对象都能定义自己"真不真"。

CPython 求真值的顺序(见 `object.c` 里的 `PyObject_IsTrue`):

1. 若对象定义了 `__bool__`,调用它,必须返回 `bool`;
2. 否则若定义了 `__len__`,调用它,`len == 0` 为假,否则为真;
3. 都没有,则对象恒为真。

```python
class Buffer:
    def __init__(self, data):
        self._data = data
    def __len__(self):
        # 没有 __bool__,于是 if 走 __len__
        return len(self._data)

buf = Buffer([])
print(bool(buf))          # False —— 长度为 0
print(bool(Buffer([1])))  # True

class Always:
    def __bool__(self):
        return False       # __bool__ 优先于 __len__

print(bool(Always()))     # False
```

这解释了为什么空容器、`0`、`0.0`、`""`、`None` 都是"假":它们要么 `__len__` 返回 0,要么 `__bool__` 返回 `False`。下面这张表列出内置的假值:

| 假值 | 原因 |
|------|------|
| `None` | 单例,`__bool__` 返回 `False` |
| `False` | 就是它自己 |
| `0`、`0.0`、`0j` | 数值零 |
| `""`、`b""` | `__len__ == 0` |
| `[]`、`()`、`{}`、`set()` | `__len__ == 0` |

底层提示:`if x:` 里没有隐式的指针判空。`None` 才是 Python 的"空指针",但它是个**真实的单例对象**(地址固定),不是 C 里的 `0` 地址。判空要写 `if x is None`,用 `is` 比较身份(即 C 里的指针相等),而不是 `==`。

## if / elif / else

语法上没有括号、靠缩进,没什么可说的。值得看的是它编译成什么字节码——纯跳转,和 C 一样:

```python
import dis

def classify(n):
    if n > 0:
        return "正"
    elif n == 0:
        return "零"
    else:
        return "负"

dis.dis(classify)
```

你会看到 `COMPARE_OP` 后跟 `POP_JUMP_IF_FALSE`——条件为假就跳到下一个分支。`elif` 没有独立的字节码,它只是 `else: if` 的语法糖,但不增加缩进层级。

Python 没有 C 那种 `? :` 三元运算符,取而代之是条件表达式:

```python
label = "正" if n > 0 else "非正"   # 值表达式,不是语句
```

## while 循环

`while` 和 C 几乎一致,条件同样走真值协议:

```python
n = 10
while n:            # 等价于 while n != 0(走 __bool__ / 数值零判断)
    n -= 1
```

`break` / `continue` 语义与 C 相同。Python **没有** `do-while`,常见替代是 `while True` + `break`。

## for 循环:与 C 的下标 for 本质不同

这是本章最重要的一节。C 的 `for` 是"初始化 - 判断 - 递增"三段,循环变量是你手里的整数,你**主动**用它去索引数组。Python 的 `for` 完全不是这样——它背后是**迭代器协议**:

```python
for x in obj:
    ...
```

等价于下面这段展开(这就是 CPython 实际做的事):

```python
_it = iter(obj)          # 调用 obj.__iter__(),拿到一个迭代器对象
while True:
    try:
        x = next(_it)    # 调用 _it.__next__()
    except StopIteration:
        break
    ...
```

对照两个协议方法:

| 方法 | 由谁实现 | 职责 |
|------|----------|------|
| `__iter__()` | 可迭代对象(iterable) | 返回一个迭代器 |
| `__next__()` | 迭代器(iterator) | 返回下一个值,耗尽时 `raise StopIteration` |

底层意义:循环的每一步都是一次**方法调用**——是消费方主动向迭代器"拉(pull)"下一个值(每步调用一次 `__next__`),而不是迭代器把值"推"给你。所以 Python 的 `for` 天生能遍历链表、树、文件行、无限序列——只要对象实现了协议,不需要下标、不需要知道长度。这也是为什么遍历时你几乎从不写索引。

手写一个迭代器,看清协议:

```python
class Countdown:
    def __init__(self, start):
        self.start = start
    def __iter__(self):
        # 返回迭代器。这里返回一个独立对象,保证可重复遍历
        return CountdownIter(self.start)

class CountdownIter:
    def __init__(self, cur):
        self.cur = cur
    def __iter__(self):
        return self          # 迭代器自身也要是可迭代的
    def __next__(self):
        if self.cur <= 0:
            raise StopIteration   # 用异常表示"结束",不是返回哨兵值
        self.cur -= 1
        return self.cur + 1

for x in Countdown(3):
    print(x)                 # 3 2 1
```

注意"可迭代对象"和"迭代器"是两个角色。`list` 是可迭代对象但不是迭代器(它没有 `__next__`);`iter(mylist)` 返回的 `list_iterator` 才是迭代器,它内部持有一个 C 层的索引 `it_index`,每次 `__next__` 递增。所以对同一个 list 可以开多个独立迭代器,各有各的游标。

```python
lst = [10, 20, 30]
a = iter(lst)
b = iter(lst)
print(next(a), next(a))   # 10 20
print(next(b))            # 10 —— b 是独立游标
```

内存图:list 只有一份数据,两个迭代器各自持有指向它的引用和一个整数游标:

```
        list_iterator a           list_iterator b
        +-----------+             +-----------+
        | it_seq  ---> [list]     | it_seq  ---> (同一 list)
        | it_index=2|      ^      | it_index=1|
        +-----------+      |      +-----------+
                           |
        list 对象  +-------+-------+
                   | ob[0]=10 (ptr)|
                   | ob[1]=20 (ptr)|
                   | ob[2]=30 (ptr)|
                   +---------------+
```

## range 是惰性对象,不是数组

C 程序员容易把 `range(1000000)` 想成"生成一个百万元素的数组"。不是。`range` 是一个**不可变的惰性序列对象**,只存 `start`/`stop`/`step` 三个整数,几乎不占内存:

```python
import sys
r = range(1_000_000)
print(sys.getsizeof(r))   # 48 字节左右,与元素个数无关
print(r[500])             # 500 —— 按需 O(1) 计算,不查表
print(len(r))             # 1000000 —— 由公式算出
```

它支持索引和 `len`(实现了序列协议),但值是**用 `start + i*step` 现算的**,从不物化成列表。这与 C 里"必须先分配数组再遍历"截然不同。要真正物化才用 `list(range(...))`。

## 推导式:有独立作用域

列表/字典/集合推导式不只是语法糖。在 Python 3 里,推导式**运行在自己独立的函数作用域**中——CPython 把推导式体编译成一个匿名函数并立即调用它:

```python
import dis

def make():
    return [i * 2 for i in range(3)]

dis.dis(make)
```

反汇编里你会看到 `MAKE_FUNCTION` + `CALL`,以及一个名为 `<listcomp>` 的内层 code 对象。后果:

```python
i = "外层"
squares = [i for i in range(3)]
print(i)          # "外层" —— 循环变量 i 不泄漏到外层!
```

这与 C 的块作用域不同,也与 Python 2 不同(那时会泄漏)。因为有独立栈帧,推导式里的循环变量、临时值都在那个内层帧的 `f_locals` 里,函数返回即销毁。

> 注意:普通 `for` 语句**没有**独立作用域,循环变量会留在当前作用域。只有推导式和生成器表达式才有。

生成器表达式更进一步,连结果列表都不物化,返回一个迭代器,惰性产出:

```python
gen = (i * 2 for i in range(1_000_000))   # 不计算任何值,不分配百万列表
print(next(gen))   # 0 —— 此刻才算第一个
```

## 循环的 else 子句

Python 独有:`for`/`while` 可带 `else`,它在循环**正常耗尽**(没有被 `break` 打断)时执行。理解成"没 break 就走 else":

```python
def find(seq, target):
    for x in seq:
        if x == target:
            print("找到", x)
            break
    else:
        # 只有循环跑完都没 break 才到这里
        print("没找到")

find([1, 2, 3], 2)   # 找到 2
find([1, 2, 3], 9)   # 没找到
```

这省掉了 C 里常见的"设一个 `found` 标志变量再判断"的写法。底层上,`break` 会跳过 `else` 块对应的字节码,正常结束则顺序落入 `else`。

## match-case:结构化模式匹配(3.10+)

`match` **不是** C 的 `switch`。C 的 `switch` 是按整数值跳转表;`match` 是**按结构解构 + 绑定**,更像函数式语言的模式匹配。它不做贯穿(fall-through),不需要 `break`。

```python
def describe(point):
    match point:
        case (0, 0):
            return "原点"
        case (0, y):                 # 捕获模式:把第二个元素绑定到 y
            return f"在 y 轴, y={y}"
        case (x, 0):
            return f"在 x 轴, x={x}"
        case (x, y) if x == y:       # guard 守卫:附加布尔条件
            return f"对角线 {x}"
        case (x, y):
            return f"一般点 ({x},{y})"
        case _:                       # 通配,类似 default
            return "不是二元组"

print(describe((0, 5)))    # 在 y 轴, y=5
print(describe((3, 3)))    # 对角线 3
```

关键:`case (0, y)` 里的 `y` 是**绑定名**,不是比较对象。裸名字**总是捕获绑定**——写 `case (0, value)` 并不会拿外层的 `value` 去比较,反而会新建一个也叫 `value` 的绑定、把外层同名变量盖掉。要拿变量的**值**去匹配,必须加 guard:`case (0, y) if y == value`;或用**带点的限定名** `case Color.RED`(点号表示这是常量引用而非绑定)。

它还能按类型和属性解构对象(类模式):

```python
class Point:
    __match_args__ = ("x", "y")      # 声明位置匹配的属性顺序
    def __init__(self, x, y):
        self.x, self.y = x, y

def quadrant(p):
    match p:
        case Point(x=0, y=0):
            return "原点"
        case Point(x, y) if x > 0 and y > 0:   # 用 __match_args__ 按位置解构
            return "第一象限"
        case Point():
            return "其他"

print(quadrant(Point(1, 2)))   # 第一象限
```

底层上,`match` 由 CPython 编译成一连串专门的匹配字节码(`MATCH_SEQUENCE`、`MATCH_CLASS`、`MATCH_MAPPING` 等)加跳转,匹配过程可能调用 `__len__`、`isinstance`、按 key 取值等,因此它比 C 的 `switch` 重得多,但表达力也强得多。

## 小结

- `if`/`while` 的条件走真值协议(`__bool__` → `__len__` → 恒真),判空用 `is None`。
- `for` 是迭代器协议(`__iter__`/`__next__`/`StopIteration`)的语法糖,值是消费方反复调用 `__next__` 主动"拉(pull)"来的,和 C 下标 for 本质不同。
- `range` 是惰性对象,只存三个整数;推导式有独立作用域,循环变量不泄漏;生成器表达式连结果都惰性化。
- 循环 `else` 在"没 break"时执行;`match-case` 是结构化解构而非整数跳转表。
