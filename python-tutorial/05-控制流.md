# 控制流

控制流的关键字看起来和 C 差不多:`if`/`while`/`for`。但底层机制差得远。C 的 `for (int i=0; i<n; i++)` 是编译期生成的整数递增 + 比较跳转;Python 的 `for` 是**运行时的方法调用协议**,每一次循环都在向一个对象发消息(`__next__`)。这一章的重点就是把这些"看似熟悉"的结构拆开,看底层到底在做什么。

## 真值测试:不是只有 0 和 1

C 里 `if (x)` 就是判断 `x != 0`(整数、指针都一样)。Python 里 `if x:` 触发的是一套**协议**,任何对象都能定义自己"真不真"。

CPython 求真值的顺序(见 `object.c` 里的 `PyObject_IsTrue`):

1. 若对象定义了 `__bool__`,调用它,必须返回 `bool`;
2. 否则若定义了 `__len__`,调用它,`len == 0` 为假,否则为真;
3. 都没有,则对象恒为真。

```python
class Buffer:
    def __init__(self, data):
        self._data = data
    def __len__(self):
        # 没有 __bool__,于是 if 走 __len__
        return len(self._data)

buf = Buffer([])
print(bool(buf))          # False —— 长度为 0
print(bool(Buffer([1])))  # True

class Always:
    def __bool__(self):
        return False       # __bool__ 优先于 __len__

print(bool(Always()))     # False
```

这解释了为什么空容器、`0`、`0.0`、`""`、`None` 都是"假":它们要么 `__len__` 返回 0,要么 `__bool__` 返回 `False`。下面这张表列出内置的假值:

| 假值 | 原因 |
|------|------|
| `None` | 单例,`__bool__` 返回 `False` |
| `False` | 就是它自己 |
| `0`、`0.0`、`0j` | 数值零 |
| `""`、`b""` | `__len__ == 0` |
| `[]`、`()`、`{}`、`set()` | `__len__ == 0` |

底层提示:`if x:` 里没有隐式的指针判空。`None` 才是 Python 的"空指针",但它是个**真实的单例对象**(地址固定),不是 C 里的 `0` 地址。判空要写 `if x is None`,用 `is` 比较身份(即 C 里的指针相等),而不是 `==`。

## if / elif / else

语法上没有括号、靠缩进,没什么可说的。值得看的是它编译成什么字节码——纯跳转,和 C 一样:

```python
import dis

def classify(n):
    if n > 0:
        return "正"
    elif n == 0:
        return "零"
    else:
        return "负"

dis.dis(classify)
```

你会看到 `COMPARE_OP` 后跟 `POP_JUMP_IF_FALSE`——条件为假就跳到下一个分支。`elif` 没有独立的字节码,它只是 `else: if` 的语法糖,但不增加缩进层级。

Python 没有 C 那种 `? :` 三元运算符,取而代之是条件表达式:

```python
label = "正" if n > 0 else "非正"   # 值表达式,不是语句
```

## while 循环

`while` 和 C 几乎一致,条件同样走真值协议:

```python
n = 10
while n:            # 等价于 while n != 0(走 __bool__ / 数值零判断)
    n -= 1
```

`break` / `continue` 语义与 C 相同。Python **没有** `do-while`,常见替代是 `while True` + `break`。

## for 循环:与 C 的下标 for 本质不同

这是本章最重要的一节。C 的 `for` 是"初始化 - 判断 - 递增"三段,循环变量是你手里的整数,你**主动**用它去索引数组。Python 的 `for` 完全不是这样——它背后是**迭代器协议**:

```python
for x in obj:
    ...
```

等价于下面这段展开(这就是 CPython 实际做的事):

```python
_it = iter(obj)          # 调用 obj.__iter__(),拿到一个迭代器对象
while True:
    try:
        x = next(_it)    # 调用 _it.__next__()
    except StopIteration:
        break
    ...
```

对照两个协议方法:

| 方法 | 由谁实现 | 职责 |
|------|----------|------|
| `__iter__()` | 可迭代对象(iterable) | 返回一个迭代器 |
| `__next__()` | 迭代器(iterator) | 返回下一个值,耗尽时 `raise StopIteration` |

底层意义:循环的每一步都是一次**方法调用**——是消费方主动向迭代器"拉(pull)"下一个值(每步调用一次 `__next__`),而不是迭代器把值"推"给你。所以 Python 的 `for` 天生能遍历链表、树、文件行、无限序列——只要对象实现了协议,不需要下标、不需要知道长度。这也是为什么遍历时你几乎从不写索引。

手写一个迭代器,看清协议:

```python
class Countdown:
    def __init__(self, start):
        self.start = start
    def __iter__(self):
        # 返回迭代器。这里返回一个独立对象,保证可重复遍历
        return CountdownIter(self.start)

class CountdownIter:
    def __init__(self, cur):
        self.cur = cur
    def __iter__(self):
        return self          # 迭代器自身也要是可迭代的
    def __next__(self):
        if self.cur <= 0:
            raise StopIteration   # 用异常表示"结束",不是返回哨兵值
        self.cur -= 1
        return self.cur + 1

for x in Countdown(3):
    print(x)                 # 3 2 1
```

注意"可迭代对象"和"迭代器"是两个角色。`list` 是可迭代对象但不是迭代器(它没有 `__next__`);`iter(mylist)` 返回的 `list_iterator` 才是迭代器,它内部持有一个 C 层的索引 `it_index`,每次 `__next__` 递增。所以对同一个 list 可以开多个独立迭代器,各有各的游标。

```python
lst = [10, 20, 30]
a = iter(lst)
b = iter(lst)
print(next(a), next(a))   # 10 20
print(next(b))            # 10 —— b 是独立游标
```

内存图:list 只有一份数据,两个迭代器各自持有指向它的引用和一个整数游标:

```
        list_iterator a           list_iterator b
        +-----------+             +-----------+
        | it_seq  ---> [list]     | it_seq  ---> (同一 list)
        | it_index=2|      ^      | it_index=1|
        +-----------+      |      +-----------+
                           |
        list 对象  +-------+-------+
                   | ob[0]=10 (ptr)|
                   | ob[1]=20 (ptr)|
                   | ob[2]=30 (ptr)|
                   +---------------+
```

## range 是惰性对象,不是数组

C 程序员容易把 `range(1000000)` 想成"生成一个百万元素的数组"。不是。`range` 是一个**不可变的惰性序列对象**,只存 `start`/`stop`/`step` 三个整数,几乎不占内存:

```python
import sys
r = range(1_000_000)
print(sys.getsizeof(r))   # 48 字节左右,与元素个数无关
print(r[500])             # 500 —— 按需 O(1) 计算,不查表
print(len(r))             # 1000000 —— 由公式算出
```

它支持索引和 `len`(实现了序列协议),但值是**用 `start + i*step` 现算的**,从不物化成列表。这与 C 里"必须先分配数组再遍历"截然不同。要真正物化才用 `list(range(...))`。

## 推导式:有独立作用域

列表/字典/集合推导式不只是语法糖:它有**自己的作用域**,循环变量不会泄漏到外层。

```python
i = "外层"
squares = [i for i in range(3)]
print(i)          # "外层" —— 循环变量 i 不泄漏到外层!
```

而"独立作用域"具体怎么实现,**分版本**:

- **Python ≤ 3.11**:把推导式体编译成一个**匿名函数并立即调用**——反汇编能看到 `MAKE_FUNCTION` + `CALL` 和一个名为 `<listcomp>` 的内层 code 对象,靠独立栈帧隔离变量。
- **Python 3.12+(PEP 709)**:list/dict/set 推导式已**内联(inlined)**到当前函数里,**不再**生成匿名函数、也没有独立栈帧或 `<listcomp>` code 对象。`dis` 里看到的是 `LOAD_FAST_AND_CLEAR` + `FOR_ITER` + `LIST_APPEND`:进入推导式时把外层同名变量"借走并清空"、结束后再**恢复**它,从而既内联(更快)又不泄漏。

```python
import dis
def make():
    return [i * 2 for i in range(3)]
dis.dis(make)     # 3.12+:无 MAKE_FUNCTION、无 <listcomp>;3.11 及更早才有
```

这与 C 的块作用域不同,也与 Python 2 不同(那时推导式变量会泄漏)。

> 注意:普通 `for` 语句**没有**独立作用域,循环变量会留在当前作用域。只有推导式和生成器表达式才有。

生成器表达式更进一步,连结果列表都不物化,返回一个迭代器,惰性产出(注意:**生成器表达式并没有被内联**,它仍编译成一个独立的 `<genexpr>` 函数,这也是它天然拥有独立作用域的原因):

```python
gen = (i * 2 for i in range(1_000_000))   # 不计算任何值,不分配百万列表
print(next(gen))   # 0 —— 此刻才算第一个
```

## 循环的 else 子句

Python 独有:`for`/`while` 可带 `else`,它在循环**正常耗尽**(没有被 `break` 打断)时执行。理解成"没 break 就走 else":

```python
def find(seq, target):
    for x in seq:
        if x == target:
            print("找到", x)
            break
    else:
        # 只有循环跑完都没 break 才到这里
        print("没找到")

find([1, 2, 3], 2)   # 找到 2
find([1, 2, 3], 9)   # 没找到
```

这省掉了 C 里常见的"设一个 `found` 标志变量再判断"的写法。底层上,`break` 会跳过 `else` 块对应的字节码,正常结束则顺序落入 `else`。

## match-case:结构化模式匹配(3.10+)

这一节我们从零讲清楚 `match`,因为它是 Python 里**最容易看着眼熟、其实和你想的不一样**的语法。

一句话:**`match` 不是 C 的 `switch`**。C 的 `switch(x)` 是"拿 x 的整数值去跳转表里查",只能比整数、还要 `break` 防贯穿;Python 的 `match` 是"看看这个值**长什么结构**,并顺手把里面的部件**取出来绑定到变量**"。所以它更像"拆快递":先看盒子形状对不对,对了就把里面的东西掏出来给你。

它**没有** fall-through(不会像 C 那样漏写 `break` 就往下贯穿),也就**不需要 `break`**:匹配上哪个 `case` 就执行哪个,然后整个 `match` 结束。

### 第 1 步:最简单的用法——按"值"匹配(这部分最像 switch)

```python
def http(code):
    match code:               # 拿 code 去和下面每个 case 比
        case 200:             # 如果 code == 200
            return "OK"
        case 404:
            return "Not Found"
        case _:               # _ 是通配符,谁都匹配,相当于 switch 的 default
            return "Unknown"

print(http(200))   # OK
print(http(404))   # Not Found
print(http(500))   # Unknown
```

- `case 200:` 这种写**字面量**(数字、字符串、`True`/`False`/`None`)时,就是"值相等就匹配",和 `switch` 直觉一致。
- `case _:` 的下划线 `_` 是**通配符**:放最后当兜底,类似 `default`。
- 从上往下**第一个**匹配的 case 生效,后面的不再看。

### 第 2 步:OR 模式——一个 case 匹配多个值

用 `|` 把多个模式并起来,任意一个中就算匹配(比 C 里堆一串 `case 400: case 401:` 干净):

```python
def kind(code):
    match code:
        case 200 | 201 | 204:      # 这三个值任意一个都算 success
            return "success"
        case 400 | 401 | 404:
            return "client error"
        case _:
            return "other"

print(kind(201))   # success
print(kind(404))   # client error
```

### 第 3 步:捕获模式——一边匹配一边"取值"(这是 `match` 的精髓)

到这里才和 `switch` 拉开差距。你可以在模式里写一个**变量名**,匹配成功时它会把对应的值**绑定(赋值)**给这个变量,后面直接用:

```python
def head_tail(seq):
    match seq:
        case []:                       # 匹配空列表
            return "空的"
        case [x]:                      # 匹配"恰好一个元素的列表",并把那个元素绑定到 x
            return f"只有一个:{x}"
        case [first, *rest]:           # 匹配"至少一个元素";first=第一个,*rest=剩下的(列表)
            return f"第一个={first},剩下={rest}"

print(head_tail([]))          # 空的
print(head_tail([9]))         # 只有一个:9
print(head_tail([1, 2, 3, 4]))# 第一个=1,剩下=[2, 3, 4]
```

- `[x]`、`[first, *rest]` 这种叫**序列模式**:它先检查"是不是序列、长度对不对",对了再把元素**拆出来**绑定到变量。`*rest` 像函数里的 `*args`,收集剩余元素。
- 元组也一样:`case (x, y):` 匹配"两个元素的元组"并把两个元素绑定到 `x`、`y`。

### ⚠️ 第 4 步:新手最大的坑——裸变量名是"绑定"不是"比较"

这是 `match` 唯一反直觉、几乎人人踩的点,单独讲清楚。

你可能想"如果 `n` 等于外面那个变量 `value` 就匹配",于是写 `case value:`。**错!** 模式里的**裸名字永远是"捕获绑定"**,不是拿它的值去比较——它会**无条件匹配**,并把 `n` 赋给一个新的 `value`,还把外层同名变量盖掉:

```python
value = 3
def bad(n):
    match n:
        case value:                    # 陷阱!这不是"n == 3",而是"把 n 绑定到 value"
            return f"绑定了 value={value}"

print(bad(99))   # 绑定了 value=99   ← 证明它根本没和外层的 3 比较,什么都能匹配
```

**想拿一个变量的值来匹配,有两种正确写法:**

```python
value = 3
def good(n):
    match n:
        case v if v == value:          # 写法一:先捕获成 v,再用 guard 比较
            return "等于 value"
        case _:
            return "不等"

# 写法二:用"带点的名字"(限定名),点号告诉 match 这是常量引用、不是绑定
import enum
class Color(enum.Enum):
    RED = 1; GREEN = 2
def name(c):
    match c:
        case Color.RED:                # 带点 → 当作值来比较(而不是绑定)
            return "红"
        case Color.GREEN:
            return "绿"
```

记住一句话:**模式里 `abc` 是"绑定",`X.abc` 才是"比较"。**

### 第 5 步:guard——给 case 加一个额外条件

在 `case` 后面加 `if 条件`,叫 **guard(守卫)**:只有"模式匹配 **且** 条件为真"才算中。

```python
def sign(n):
    match n:
        case 0:
            return "零"
        case x if x > 0:        # 先捕获成 x,再要求 x > 0
            return "正"
        case _:
            return "负"

print(sign(0), sign(5), sign(-2))   # 零 正 负
```

### 第 6 步:映射模式(匹配 dict)与类模式(匹配对象)

`match` 还能拆字典和自定义对象——这在处理 JSON、AST、事件时非常好用。

**映射模式**:只关心"有没有某些键、值是什么",多余的键忽略:

```python
def route(req):
    match req:
        case {"method": "GET", "path": p}:   # 要求有 method=="GET",并把 path 的值绑定到 p
            return f"GET {p}"
        case {"method": "POST"}:             # 只要求有 method=="POST",别的键不管
            return "POST"
        case _:
            return "未知请求"

print(route({"method": "GET", "path": "/users"}))   # GET /users
```

**类模式**:按类型 + 属性解构对象。给类加一个 `__match_args__`,就能像元组一样按**位置**写:

```python
class Point:
    __match_args__ = ("x", "y")     # 声明:位置匹配时,第1个对应 x、第2个对应 y
    def __init__(self, x, y):
        self.x, self.y = x, y

def quadrant(p):
    match p:
        case Point(0, 0):                    # 位置匹配:x==0 且 y==0
            return "原点"
        case Point(x=0, y=yy):               # 关键字匹配:x==0,并把 y 绑定到 yy
            return f"在 y 轴,y={yy}"
        case Point(x, y) if x > 0 and y > 0: # 位置捕获 + guard
            return "第一象限"
        case Point():                        # 只要是 Point 就行
            return "其他 Point"
        case _:
            return "根本不是 Point"

print(quadrant(Point(0, 0)))   # 原点
print(quadrant(Point(0, 7)))   # 在 y 轴,y=7
print(quadrant(Point(1, 2)))   # 第一象限
```

### 小抄:常见模式一览

| 模式 | 例子 | 含义 |
|------|------|------|
| 字面量 | `case 200:` | 值相等 |
| 通配符 | `case _:` | 匹配任何东西(兜底) |
| 捕获 | `case x:` | **总匹配**,并把值绑定到 `x` |
| OR | `case 1` \| `2` | 任一子模式匹配 |
| 序列 | `case [a, b, *rest]:` | 是序列且解构 |
| 映射 | `case {"k": v}:` | 是字典且含键 `k` |
| 类 | `case Point(x, y):` | 是该类型且按属性解构 |
| guard | `case x if x > 0:` | 模式匹配**且**条件成立 |
| 限定名比较 | `case Color.RED:` | 拿 `Color.RED` 的值去比较(非绑定) |

### 底层:它到底做了什么

`match` 由 CPython 编译成一连串**专门的匹配字节码**(`MATCH_SEQUENCE`、`MATCH_MAPPING`、`MATCH_CLASS` 等)加跳转;匹配过程可能会调用 `__len__`、`isinstance`、按 key/属性取值等。所以它比 C 的 `switch`(一条跳转表指令)**重得多**——但换来的是对**结构**的表达力,这是 `switch` 给不了的。用 `import dis; dis.dis(describe)` 可以看到这些指令。

> 一句话总结 `match`:**字面量按值比,裸名字是绑定,`|` 是或,`[]`/`{}`/`类()` 按结构拆,`if` 加条件,`_` 兜底。**

## 小结

- `if`/`while` 的条件走真值协议(`__bool__` → `__len__` → 恒真),判空用 `is None`。
- `for` 是迭代器协议(`__iter__`/`__next__`/`StopIteration`)的语法糖,值是消费方反复调用 `__next__` 主动"拉(pull)"来的,和 C 下标 for 本质不同。
- `range` 是惰性对象,只存三个整数;推导式有独立作用域,循环变量不泄漏;生成器表达式连结果都惰性化。
- 循环 `else` 在"没 break"时执行;`match-case` 是结构化解构而非整数跳转表。
