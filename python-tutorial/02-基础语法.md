# 基础语法：从大括号世界过来的适应指南

你已经能读懂 [第01章](01-运行模型.md) 的运行模型，知道每个值都是堆对象。这一章处理表层语法：那些让你第一眼觉得"不对劲"的东西——没有大括号、没有分号、没有类型声明、没有 `main`。表面差异背后，往往对应着 C/C++ 里被省掉或改写的机制。

## 一、缩进即块：语法层强制的作用域

C/C++ 用 `{}` 界定块，缩进纯属美观，编译器不看：

```c
if (x > 0) {
    foo();
    bar();
}
```

Python 用**缩进本身**界定块，没有大括号，语句以换行结束（无分号）：

```python
if x > 0:
    foo()
    bar()
# 回退缩进即出块
```

规则：
- 冒号 `:` 引导一个块，块体必须缩进（约定 4 空格）。
- 同一块内所有语句缩进必须严格一致，否则 `IndentationError`。
- **不要混用 Tab 和空格**——CPython 会直接报错。配置编辑器把 Tab 转空格。

底层视角：缩进在词法分析阶段被转成 `INDENT` / `DEDENT` token（回顾 [第01章](01-运行模型.md) 的 tokenize 阶段），它们扮演的角色正是 C 里的 `{` 和 `}`。所以缩进不是"风格建议"，它是**语法记号**，跟你少写一个大括号导致编译错误是一回事。

一个重要的语义差异：C 的 `{}` 会引入新的**词法作用域**（块内声明的变量出块即失效）。Python 的缩进块**不引入新作用域**——`if`/`for`/`while` 块里绑定的名字，出块后依然可见：

```python
if True:
    y = 10
print(y)   # 10，合法！y 没有"出块销毁"
```

Python 里真正引入新作用域的只有**函数、类、模块、推导式**，不是每个缩进块。作用域细节见 [第06章](06-函数与调用栈.md)。

## 二、动态类型、强类型、无需声明

三个正交的概念，别混：

- **无需声明**：变量不预先声明类型，第一次赋值即"诞生"。`x = 3` 就够了，没有 `int x`。
- **动态类型**：类型属于**对象**，不属于变量。同一个名字可以先指 int 再指 str。类型检查发生在运行期（回顾 [第01章](01-运行模型.md)：`ob_type` 挂在对象上）。
- **强类型**：不做隐式的乱来转换。`1 + "2"` 直接抛 `TypeError`，不像弱类型语言给你个 `"12"` 或 `3`。

```python
x = 42          # 现在 x 指向一个 int 对象
x = "hello"     # 现在 x 指向一个 str 对象；完全合法
x = [1, 2, 3]   # 又指向 list 对象

# 强类型：不隐式转换
# 1 + "2"       # TypeError: unsupported operand type(s)
1 + int("2")    # 3，显式转换才行
```

对照 C++：C++ 是静态 + 强类型，`int x` 一旦定下，`x = "hi"` 编译不过。Python 把类型检查推迟到运行时，换来灵活性，代价是很多错误编译期发现不了（可用类型注解 + `mypy` 补救，见 [第08章](08-面向对象与类型系统.md)）。

注意"变量重新赋值"底层是什么：`x = "hello"` **不是**把 int 对象的内存改写成字符串，而是让名字 `x` 改指向一个新的堆对象，旧的 int 对象引用计数减一可能被回收。这条是全书重点，[第03章](03-变量与内存模型.md) 会用内存图讲透。

## 三、没有 main：模块从上到下执行

C/C++ 程序入口是 `main`。Python 没有强制入口——**模块被加载时，顶层语句从上到下依次执行**。函数定义 `def` 本身也是一条"执行时创建函数对象"的语句，不是编译期声明。

那怎么区分"作为脚本直接运行"和"被别人 import"？靠这个惯用法：

```python
def main():
    ...

if __name__ == "__main__":   # 直接运行时 __name__ == "__main__"
    main()
```

被 `import` 时，该模块的 `__name__` 是模块名，条件为假，`main()` 不执行——于是导入它只拿到函数定义而不触发副作用。这是"库/脚本两用"的标准写法，功能上对应 C 的 `int main()` 入口，但机制完全不同：它只是个普通的运行期 `if`。

## 四、print / input 与注释

```python
print("hello", "world")          # 默认空格分隔、末尾换行 -> hello world
print("no newline", end="")      # 自定义结尾
print(a, b, sep=", ")            # 自定义分隔符
name = input("name? ")           # 读一行，去掉换行；返回值永远是 str

# 注释
# 单行注释用井号
"""三引号字符串常被当多行注释用，
但它其实是个字符串字面量对象。"""
```

`print` 是函数（Python 3 起），不是语句。`input` 返回 `str`，要数字得自己 `int(input())`——没有 `scanf("%d")` 那种格式化读取。

## 五、运算符：几个你没见过的

大部分运算符（`+ - * / % == < > && ||`）语义你熟，但 Python 用关键字 `and` / `or` / `not` 代替 `&& || !`（`& | ^ ~ << >>` 仍是按位运算）。几个陌生的：

| 运算符 | 含义 | C/C++ 类比 |
|--------|------|-----------|
| `/` | 真除法，**总返回 float** | `5/2 == 2.5`（不是 C 的整除 2！） |
| `//` | 向下取整除法 | 类似 C 整数除法但向负无穷取整 |
| `**` | 幂 | 无对应，得用 `pow()` |
| `@` | 矩阵乘法 | 无；由对象 `__matmul__` 定义，NumPy 用 |
| `and`/`or` | 逻辑与或（短路） | `&&` / `\|\|` |

务必记住 `/` 的坑：

```python
5 / 2     # 2.5  （C 里 5/2 == 2，这里不是！）
5 // 2    # 2
-5 // 2   # -3   （向下取整，向负无穷；C 里 -5/2 == -2 向零截断）
5 % 2     # 1
-5 % 2    # 1    （结果符号跟除数，跟 C 不同！C 里 -5%2 == -1）
2 ** 10   # 1024
```

`//` 和 `%` 的取整/符号规则与 C 不同，这在移植数值代码时是经典 bug 来源。Python 保证 `a == (a // b) * b + (a % b)`，且余数符号随除数。

整数还有个大杀器：**任意精度**。Python 的 `int` 不会溢出，`2 ** 200` 直接算，因为它是堆上的变长对象（[第01章](01-运行模型.md) 提过 `PyLongObject` 头后跟变长数字位）。没有 `int32` 溢出回绕这回事。

```python
x = 2 ** 200          # 不溢出,直接算出全部 61 位十进制数字
print(x)              # 1606938044258990275541962092341162602522202993782792835301376
print(type(x))        # <class 'int'>  始终是 int,没有 int64 上限
print(x.bit_length()) # 201
```

## 六、f-string：格式化字符串

```python
name, n = "Ann", 3
print(f"{name} has {n} apples")        # Ann has 3 apples
print(f"{n} squared is {n**2}")        # 花括号里可放任意表达式
print(f"pi = {3.14159:.2f}")           # pi = 3.14  （:.2f 格式说明）
print(f"{n=}")                         # n=3        （调试用，自动带变量名）
print(f"{255:#x}")                     # 0xff
```

f-string 在编译期就被拆成字符串拼接的字节码（可用 `dis` 验证），运行时高效，是首选格式化方式。

亲手验证"编译期拆解"：没有运行时解析格式串这回事，编出来就是"逐段格式化 + 拼接"。

```python
import dis
name, n = "Ann", 3
dis.dis('f"{name} has {n}"')
```

输出（3.14）：每个 `{}` 对应一条 `FORMAT_SIMPLE`，最后 `BUILD_STRING 3` 把三段（`name`、`' has '`、`n`）拼成一个 str：

```
  0           RESUME                   0

  1           LOAD_NAME                0 (name)
              FORMAT_SIMPLE
              LOAD_CONST               0 (' has ')
              LOAD_NAME                1 (n)
              FORMAT_SIMPLE
              BUILD_STRING             3
              RETURN_VALUE
```
它取代了老的 `%` 格式化和 `str.format()`。类比 C 的 `printf`/`snprintf`，但表达式直接内嵌、类型自动处理，无需 `%d`/`%s` 占位符匹配。

## 七、布尔、None 与真值测试

```python
flag = True          # 首字母大写！True / False，不是 true / false
nothing = None       # 空值单例，类比 nullptr / NULL（但它是个对象）
```

- `True` / `False` 是 `bool` 类型；在 CPython 中它们其实是 `int` 的子类，`True == 1`、`False == 0`。
- `None` 是全局唯一单例对象，表示"无值"，类比 `nullptr`——但它是个正儿八经的堆对象，判等要用 `is`：`if x is None:`（不是 `== None`）。

**真值测试（truthiness）**：任何对象都能放进 `if`，不止 bool。C 里"非零即真"，Python 更丰富：

| 假值（falsy） | 真值（truthy） |
|--------------|---------------|
| `False`、`None` | 其余一切 |
| `0`、`0.0`、`0j` | 非零数字 |
| `""`、`[]`、`{}`、`()`、`set()` | 非空容器 |

```python
items = []
if items:                # 空列表为假；不用写 len(items) == 0
    print("有货")
else:
    print("空的")        # 走这里
```

底层：`if x` 会调用对象的 `__bool__`（无则退回 `__len__`，判是否为 0）。这是运行期通过 `ob_type` 的方法分派，跟 [第08章](08-面向对象与类型系统.md) 的协议一脉相承。

## 八、链式比较与多重赋值

**链式比较**——C 里绝对没有，也是 C 移植者最容易踩的坑：

```python
x = 5
if 0 < x < 10:           # 读作 (0 < x) and (x < 10)
    print("范围内")

# C 里 0 < x < 10 会被解析成 (0 < x) < 10，即 (true) < 10 == 1 < 10 == true，永远真！
# Python 不会，它真的是数学意义上的链式
```

**多重赋值 / 解包**：

```python
a, b = 1, 2              # 同时绑定
a, b = b, a             # 交换，无需临时变量
x = y = z = 0           # 三个名字都指向同一个 0 对象（注意别名，见第03章）
first, *rest = [1, 2, 3, 4]   # first=1, rest=[2,3,4]，星号收集剩余
```

`a, b = b, a` 的底层：右侧两个值先各自求值压栈，再分别绑定给左侧，所以不需要临时变量，也不存在求值顺序陷阱。CPython 会把小规模交换优化掉：**3.11–3.12** 用一条 `SWAP` 指令(`SWAP 2`)；**3.13+** 进一步用超级指令 `LOAD_FAST_LOAD_FAST` + `STORE_FAST_STORE_FAST`(连 `SWAP` 都省了)。只有更早版本或元素更多时才走 `BUILD_TUPLE` + `UNPACK_SEQUENCE`。本机 3.14 亲测：

```python
import dis

def swap(a, b):
    a, b = b, a

dis.dis(swap)
```

输出印证了"连 `SWAP`/元组打包都省了"——一条超级指令读、一条超级指令写：

```
  3           RESUME                   0

  4           LOAD_FAST_LOAD_FAST     16 (b, a)
              STORE_FAST_STORE_FAST   16 (b, a)
              LOAD_CONST               0 (None)
              RETURN_VALUE
```

## 九、C/C++ ↔ Python 语法差异速查表

| 主题 | C/C++ | Python |
|------|-------|--------|
| 块界定 | `{ }` | 缩进（INDENT/DEDENT） |
| 语句结束 | `;` | 换行 |
| 变量声明 | `int x;` 必须 | 直接 `x = ...` |
| 类型 | 静态、绑在变量 | 动态、绑在对象 |
| 入口 | `main()` | 顶层语句；`if __name__=="__main__"` |
| 逻辑运算 | `&& \|\| !` | `and or not` |
| 除法 `/` | 整数间截断取整 | 总是 float，整除用 `//` |
| 取模符号 | 随被除数 | 随除数 |
| 整数溢出 | 有（回绕/UB） | 无，任意精度 |
| 三目 | `c ? a : b` | `a if c else b` |
| `++`/`--` | 有 | **没有**，用 `x += 1` |
| 空值 | `NULL`/`nullptr` | `None`（对象，用 `is` 判） |
| 布尔字面量 | `true`/`false` | `True`/`False` |
| 自增自减 | `i++` | 不存在 |
| switch | `switch/case` | `match/case`（3.10+，见第05章） |
| 常量 | `const` | 无强制，靠命名约定（全大写） |

补充两个易忘点：
- **没有 `++`/`--`**。`x++` 是语法错误，用 `x += 1`。
- 三目运算符是 `值1 if 条件 else 值2`，顺序跟 C 不一样：`y = a if x > 0 else b`。

## 十、小结

- 缩进是**语法记号**（编译期转成 INDENT/DEDENT），代替大括号；但缩进块不引入新作用域，只有函数/类/模块**以及推导式、生成器表达式**才引入。
- 动态强类型、无需声明：类型绑在对象上、运行期检查，但不做隐式转换。
- 没有 main，模块顶层顺序执行；`if __name__ == "__main__"` 区分脚本与库。
- 运算符坑：`/` 恒为 float、`//` 向下取整、`%` 符号随除数、整数任意精度、没有 `++`。
- 真值测试比 C 丰富（空容器为假）；链式比较是真链式；`a, b = b, a` 由 CPython 优化实现（3.11–3.12 是 `SWAP`，3.13+ 是 `LOAD_FAST_LOAD_FAST`/`STORE_FAST_STORE_FAST`，更早才是元组打包解包）。
