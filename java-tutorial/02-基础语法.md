# 基础语法：与 C/C++ 对照

这一章快速铺平语法层面的差异，让你能读懂代码。凡是和 C/C++ 一样的（`{}` 块、`;` 结尾、`//` `/* */` 注释、`+ - * /`、`&& || !`、`if/for/while`）不再赘述；重点讲**结构性差异**和背后的取舍。

## 一、一切都在 class 里；程序入口

C 有自由函数，`main` 是全局函数。Java **没有自由函数、没有全局变量**——所有代码都必须挂在某个类（或接口/枚举/记录）里。最小可运行程序：

```java
public class App {
    // 程序入口。签名必须是 public static void main(String[])
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
```

对照 C 的 `int main(int argc, char** argv)`：
- **无返回值**：`main` 返回 `void`。要设置进程退出码用 `System.exit(int)`（对应 C 的 `exit()`）。
- **参数是 `String[] args`**：只有真正的参数，**不含程序名本身**（C 的 `argv[0]` 是程序名，Java 里 `args[0]` 已是第一个真实参数）。`argc` 也不需要——数组自带 `length`。
- **`static`**：JVM 启动时还没有任何对象实例，只能调用不依赖实例的静态方法，所以入口必须 `static`。见[「面向对象与类型系统」](08-面向对象与类型系统.md)中 static 的含义。

> JDK 21 起以预览形式提供更简的入口（省略 class 包裹、实例 `main`），并在 **JDK 25 转为正式特性**（JEP 512，不再是预览）；但为跨版本兼容，正式项目多数仍用上述经典写法。

## 二、包与 import：没有头文件、没有预处理器

这是与 C/C++ 差异最大、也最容易让你舒服的地方。

C/C++ 的模型：`#include` 做**文本粘贴**，头文件声明 + 源文件定义分离，靠链接器拼接；`#define` 宏在预处理期做文本替换；容易出现重复包含、宏污染、编译单元膨胀。

Java 的模型：
- **包 package**：命名空间，对应目录结构。`package com.example.util;` 意味着该文件在 `com/example/util/` 下。
- **import**：只是**告诉编译器某个简单名对应哪个全限定名**，它**不做任何文本插入、不引入代码**。`import java.util.List;` 之后你能写 `List` 而非 `java.util.List`。编译和链接是按类粒度、由类加载器在运行时解析的（见[「运行模型」](01-运行模型.md)）。

```java
package com.example;

import java.util.List;          // 引入单个类
import java.util.*;             // 引入整个包（不递归子包）
import static java.lang.Math.*; // 静态导入：之后可直接写 sqrt(2) 而非 Math.sqrt(2)

public class Demo { /* ... */ }
```

**没有的东西**（以及原因）：
- **没有头文件**：声明与实现都在同一个 `.class` 里，`.class` 自带完整签名信息，编译器直接读取，不需要你维护 `.h`。
- **没有预处理器 / 宏**：没有 `#define`、`#ifdef`、条件编译。常量用 `static final`，"函数宏"用普通方法（反正会被 JIT 内联，见[「运行模型」](01-运行模型.md)），平台差异靠运行时判断或构建工具处理。少了宏，也就少了宏带来的一整类 bug。
- **没有前向声明**：类之间可以互相引用、循环依赖，编译器一遍扫描能解析，无需像 C++ 那样操心声明顺序。

## 三、强静态类型：必须声明；`var` 局部推断

和 C++ 一样是**强静态类型**：每个变量有编译期确定的类型，类型不匹配编译报错。

```java
int count = 10;
double ratio = 0.5;
String name = "Ada";
long big = 9_000_000_000L;   // 超 int 范围要加 L 后缀
```

JDK 10 起支持 `var` 做**局部变量类型推断**，语义等同 C++11 的 `auto`：

```java
var count = 10;              // 推断为 int
var name = "Ada";            // 推断为 String
var list = new ArrayList<String>();  // 推断为 ArrayList<String>
```

关键限制与澄清：
- **`var` 只用于局部变量**（方法内、含有初始化）。不能用于字段、方法参数、返回类型。C++ 的 `auto` 适用范围更广。
- **它不是动态类型**：类型在编译期就定死了，`var` 只是省去你手写类型名。运行期没有任何区别，字节码里就是具体类型。
- 不能 `var x;` 不初始化（无从推断），也不能 `var x = null;`（null 无类型）。

## 四、运算符

绝大多数与 C 相同：算术 `+ - * / %`、关系 `== != < > <= >=`、逻辑 `&& || !`、位运算 `& | ^ ~ << >>`、复合赋值 `+=` 等、自增自减 `++ --`。差异点：

- **无符号右移 `>>>`**：Java 没有 unsigned 类型（见[「数据类型」](04-数据类型.md)），于是补了个 `>>>` 做逻辑右移（高位补 0），`>>` 是算术右移（补符号位）。C 里是否补符号取决于类型是否 unsigned；Java 用运算符区分。
- **`+` 用于字符串拼接**：`"a" + 1` 得到 `"a1"`。这是唯一被"重载"的运算符，且由编译器特殊处理（见[「数据类型」](04-数据类型.md)的 StringBuilder 说明）。
- **`instanceof`**：运行时类型判断（C++ 的 `dynamic_cast` 味道），还支持模式绑定，见[「控制流」](05-控制流.md)。
- **整数除零**：`1/0` 抛 `ArithmeticException`（不是 UB！）；但**浮点**除零得到 `Infinity`/`NaN`，与 IEEE754 一致。C 里整数除零是未定义行为。
- **移位计数取模**：`int` 移位时位移量按 `& 31` 处理，`long` 按 `& 63`。`1 << 32` 结果是 `1` 而非 0，别踩坑。
- **无逗号运算符**，`for` 头里的逗号是语法特例。
- **布尔严格**：`if` 条件必须是 `boolean`，`if (x)` 里 x 不能是 int。C 里"非零即真"在 Java 不成立，从根上杜绝 `if (a = b)` 笔误。

## 五、字符串

`String` 是**类**，不是 `char*`，也不是 `char[]`。它**不可变（immutable）**，内容存在堆上（内部是 `byte[]` + 编码标记）。

```java
String s = "hello";
int n = s.length();               // 不是 strlen；不靠 \0 结尾
char c = s.charAt(0);
boolean same = s.equals("hello"); // 比内容用 equals，不是 ==！
String up = s.toUpperCase();      // 返回新串，s 本身不变
String cat = s + " world";        // 拼接
String f = String.format("%d-%s", 42, "x");  // 类 printf
```

对 C 程序员的三个必记点：
1. **不以 `\0` 结尾**，长度是显式字段，`length()` 是 O(1)。
2. **比较内容用 `.equals()`，不要用 `==`**。`==` 比的是引用（是不是同一个对象），这是新手头号陷阱，[「变量与内存模型」](03-变量与内存模型.md)会从内存层面讲透。
3. **不可变**：任何"修改"都返回新对象。频繁拼接用 `StringBuilder`（类似手动管理的可变缓冲区），细节见[「数据类型」](04-数据类型.md)。

## 六、控制台 IO

```java
// 输出
System.out.println("整行带换行");
System.out.print("不带换行");
System.out.printf("%d 和 %.2f%n", 10, 3.14159);  // 类 printf，%n 是平台换行

// 输入
import java.util.Scanner;
Scanner sc = new Scanner(System.in);
int x = sc.nextInt();
String line = sc.nextLine();
```

`System.out` 是一个 `PrintStream` 对象（对应 stdout），`System.in` 是 `InputStream`（stdin），`System.err` 是 stderr。没有 `printf/scanf` 那套可变参格式的类型不安全问题——`printf` 在 Java 里做运行时检查，类型不符抛异常而非静默 UB。

## 七、注释与 Javadoc

```java
// 单行
/* 多行 */

/**
 * Javadoc 文档注释，用 javadoc 工具生成 HTML API 文档。
 * @param n 输入数量
 * @return 计算结果
 */
public int compute(int n) { return n * 2; }
```

Javadoc 相当于 Doxygen 内置进工具链，`@param`/`@return`/`@throws` 等标签是约定。IDE 也据此显示悬停文档。

## 八、与 C++ 差异速查表

| 维度 | C++ | Java |
|------|-----|------|
| 继承 | 多继承 | **单继承类** + 多实现接口（见[第 8 章](08-面向对象与类型系统.md)） |
| 指针 | 有指针语法 `*`/`&`/`->` | **无显式指针语法**；引用是隐式受管指针 |
| 内存释放 | `delete`/`free` 手动 | **无**，GC 自动回收 |
| 运算符重载 | 支持 | **不支持**（除内置 `+` 拼接字符串） |
| 自由函数 | 支持 | **无**，一切在类里（可用 static 方法） |
| 数组 | 栈/堆上的连续内存，无边界检查 | **是对象**，在堆上，带 `length`，**有边界检查** |
| 头文件 | 需要 | **无**，无预处理器/宏 |
| 默认参数 | 支持 | **不支持**，用方法重载替代（见[第 6 章](06-函数与调用栈.md)） |
| 值/引用语义 | 对象可值语义（拷贝构造）、可引用 | 对象**只有引用语义**；基本类型值语义 |
| 析构 | RAII 确定性析构 | 无确定性析构；`finalize` 已废弃，资源用 try-with-resources（见[第 9 章](09-错误处理与资源管理.md)） |
| 栈上对象 | `Foo f;` 直接栈上 | 语言层永远 `new` 到堆；HotSpot 逃逸分析可能做**标量替换**（把对象拆成标量放寄存器/栈帧，而非整体搬上栈） |
| `const` | const 修饰广泛 | `final`（语义不完全等价，见[第 3 章](03-变量与内存模型.md)） |

几点强调：
- **数组是对象**：`int[] a = new int[10];`，它在堆上，`a.length` 是字段，越界抛 `ArrayIndexOutOfBoundsException` 而不是踩内存。这从根上消除了 C 数组越界这类最危险的 bug，代价是每次访问的边界检查（JIT 常能消除循环内的冗余检查）。
- **无手动 delete**：你 `new` 完就不管了，GC 负责回收。但**非内存资源**（文件、连接）仍需显式关闭——见[第 9 章](09-错误处理与资源管理.md)。
- **无运算符重载**：`a.equals(b)`、`a.compareTo(b)`、`bigInt.add(other)` 都是方法调用，不能写成 `a == b`（那比引用）或 `a + b`。这让代码更啰嗦但无歧义。

## 小结

- 代码全在类里，入口是 `public static void main(String[])`，无自由函数、无全局量。
- 包 + import 取代头文件与预处理器；import 不插代码，只解析名字。
- 强静态类型需声明，`var` 提供局部类型推断（编译期定死，非动态）。
- 运算符大体同 C，注意 `>>>`、字符串 `+`、除零抛异常、`if` 必须 boolean。
- `String` 是不可变的堆对象，比内容用 `equals`。
- 相对 C++：单继承、无指针语法、无手动释放、无运算符重载、数组是带边界检查的对象。
