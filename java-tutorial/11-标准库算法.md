# 标准库算法

C++ 的 `<algorithm>`、`std::sort`(introsort)、`std::set`(红黑树)、`priority_queue`(堆)在 STL 里。Java 对应的是 `java.util.Collections`、`Arrays`、`PriorityQueue`、`TreeMap`,加上函数式的 Stream API。

**本章每一节都是先给一段能直接照抄运行的代码,把"怎么用"讲清楚,再讲"为什么这么设计、底层怎么实现"。你不需要先懂 JVM 内部结构或字节码,跟着每节开头的代码抄改就能开始用这些工具;想搞清楚"为什么 `Arrays.sort` 对 `int[]` 和 `Integer[]` 用不同算法"这类问题,再往后看原理段落即可。**

## Collections 工具类

`java.util.Collections` 是一堆静态方法,操作 `List`/`Collection`:

```java
List<Integer> list = new ArrayList<>(List.of(3, 1, 2));
Collections.sort(list);                          // 原地排序(内部转数组用 TimSort)
int idx = Collections.binarySearch(list, 2);     // 已排序 List 上二分;链表(非 RandomAccess)是 O(n) 次链表遍历 + O(log n) 次比较
Collections.reverse(list);
Collections.shuffle(list);
int max = Collections.max(list);

// 不可变视图:返回包装,任何修改抛 UnsupportedOperationException(类比 const 引用,但运行期检查)
List<Integer> ro = Collections.unmodifiableList(list); // 底层仍指向 list,list 变它也变
List<Integer> imm = List.of(1, 2, 3);            // JDK9+ 真正不可变副本
```

> `binarySearch` 要求 List 已按同一顺序排序,否则结果未定义。返回负数 `-(插入点)-1` 表示未找到,可用于求插入位置。

## Arrays.sort:两种算法

这是本章重点。`Arrays.sort` 对**基本类型**和**对象数组**用不同算法,这不是随意为之:

```java
int[] a = {5, 2, 8, 1};
Arrays.sort(a);                    // Dual-Pivot Quicksort(双轴快排)

Integer[] b = {5, 2, 8, 1};
Arrays.sort(b);                    // TimSort(归并变体)
Arrays.sort(b, Comparator.reverseOrder());
```

### 为何区别对待

| 维度 | 基本类型 int[] | 对象 Object[] |
|------|--------------|--------------|
| 算法 | 双轴快排 | TimSort |
| 稳定性 | 无所谓(值相等即完全相同) | 必须稳定 |
| 比较代价 | 极低(机器指令) | 高(调 compareTo/compare,可能虚方法) |
| 关注点 | 缓存局部性、比较次数 | 减少比较次数、稳定性 |

**基本类型用双轴快排的理由**:两个 `int` 相等就是同一个值,交换它们无可观测差别,所以**稳定性无意义**。快排原地、缓存友好(顺序访问局部性好),比较是廉价的 CPU 指令。双轴快排选两个 pivot 把数组分三段,比经典单轴快排**减少约 5% 比较**、缓存局部性更好——注意它的**交换次数其实比单轴更多**,优势来自"比较更少 + 访存友好",而非"少移动"。

**对象用 TimSort 的理由**:
1. **稳定性**:对象排序常按某字段,相等元素的原相对顺序必须保留(比如先按 A 排、再按 B 排,B 相等时要维持 A 的顺序)。快排不稳定,TimSort(归并类)稳定。
2. **比较昂贵**:对象比较是方法调用(可能是接口虚调用),TimSort 在近乎有序的数据上比较次数接近 O(n),而快排最坏 O(n²)。真实数据常局部有序,TimSort 识别已有序的"run"并合并,大幅减少比较。

> TimSort 结合归并排序与插入排序:扫描出递增/递减的 run,短 run 用二分插入排序扩展到 minrun 长度,再按栈上不变式归并。最坏 O(n log n),已有序 O(n)。它需要 O(n) 辅助空间(归并),而快排原地——对象数组多一层指针间接,缓存本就差,额外空间可接受。

## Comparable 与 Comparator

- `Comparable<T>`:**自然顺序**,类自己实现 `compareTo`,一个类只有一种。类比 C++ 的 `operator<`。
- `Comparator<T>`:**外部策略**,可定义任意多种,类比传给 `std::sort` 的仿函数。

下面例子用到两个可能没见过的语法:`record` 是 Java 16+ 的简写,一行声明一个只存数据的不可变类(类比 C++ 里一个只有字段、自动生成构造函数和 getter 的 `struct`);`Person::name` 是**方法引用**,是 `p -> p.name()` 的简写形式,可以直接理解成"取 name 字段的 lambda"。

```java
record Person(String name, int age) implements Comparable<Person> {
    @Override public int compareTo(Person o) {   // 自然顺序:按年龄
        return Integer.compare(this.age, o.age);  // 勿用 this.age-o.age,溢出风险
    }
}

// Comparator 组合器(JDK8+),声明式,类似链式 SQL ORDER BY
Comparator<Person> cmp = Comparator
    .comparing(Person::name)                     // 先按名字
    .thenComparingInt(Person::age)               // 名字相同再按年龄
    .reversed();                                 // 整体反转

Comparator<Person> byAgeNullsFirst = Comparator
    .comparingInt(Person::age)
    .reversed();
List<Person> people = new ArrayList<>();
people.sort(cmp);   // List.sort 内部走 TimSort
```

契约:`compare` 须满足自反、传递、反对称;且与 `equals` 一致性建议保持(否则 `TreeMap`/`TreeSet` 行为反直觉)。违反传递性会让 TimSort 抛 `IllegalArgumentException: Comparison method violates its general contract`。

可运行:先按 `name` 升序,`name` 相同再按 `age` 降序——`thenComparing` + `reversed` 声明式组合:

```java
import java.util.*;

public class CmpDemo {
    record Person(String name, int age) {}

    public static void main(String[] args) {
        List<Person> people = new ArrayList<>(List.of(
            new Person("Bob", 30), new Person("Ann", 30),
            new Person("Bob", 25), new Person("Ann", 20)));

        // 先按 name 升序,name 相同再按 age 降序
        people.sort(Comparator.comparing(Person::name)
                              .thenComparing(Comparator.comparingInt(Person::age).reversed()));

        for (Person p : people) System.out.println(p.name() + " " + p.age());
    }
}
// Ann 30
// Ann 20
// Bob 30
// Bob 25
```

## PriorityQueue:二叉堆

`PriorityQueue` 是**基于数组的小顶二叉堆**(默认最小元素在队首),类比 C++ `priority_queue`(默认大顶)。

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();      // 小顶堆
pq.offer(3); pq.offer(1); pq.offer(2);
pq.peek();     // 1,查看堆顶 O(1)
pq.poll();     // 1,弹出并下沉调整 O(log n)

// 大顶堆 / 自定义
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
```

内部 `Object[] queue`,父子索引关系:节点 `i` 的孩子是 `2i+1`、`2i+2`。`offer` 上浮(siftUp),`poll` 把末尾元素移到堆顶再下沉(siftDown)。

```
数组 [1,3,2,7,4] 表示的堆:
          1(0)
        /     \
      3(1)     2(2)
      /  \
    7(3) 4(4)
```

**非线程安全**,也**不保证遍历有序**(只有堆顶有序);要有序遍历得反复 `poll`。并发场景用 `PriorityBlockingQueue`。

可运行:用大小为 k 的**小顶堆**求 Top-K 最大值——堆顶恰是这 k 个里最小的,超出就弹掉,单次 `O(log k)`:

```java
import java.util.*;

public class HeapTopK {
    public static void main(String[] args) {
        int[] data = {5, 1, 9, 3, 7, 8, 2, 6, 4};
        int k = 3;

        // 维护 k 个最大值:小顶堆的堆顶 = 当前 top-k 中最小者
        PriorityQueue<Integer> pq = new PriorityQueue<>();  // 小顶堆
        for (int x : data) {
            pq.offer(x);
            if (pq.size() > k) pq.poll();                   // 弹掉最小,O(log k)
        }

        // 逐个 poll:小顶堆按升序出队
        List<Integer> topK = new ArrayList<>();
        while (!pq.isEmpty()) topK.add(pq.poll());
        System.out.println("top-" + k + " ascending = " + topK);
    }
}
// top-3 ascending = [7, 8, 9]
```

## TreeMap / TreeSet:红黑树

`TreeMap` 底层是**红黑树**(自平衡 BST),`TreeSet` 是其包装(值当 key)。对应 C++ `std::map`/`std::set`。所有操作 O(log n),且**按 key 有序**。

```java
TreeMap<String, Integer> tm = new TreeMap<>();
tm.put("banana", 2); tm.put("apple", 1); tm.put("cherry", 3);
// 有序视图与导航方法(HashMap 没有)
tm.firstKey();               // "apple"
tm.floorKey("b");            // "apple"(≤"b" 的最大 key)
tm.ceilingKey("b");          // "banana"(≥"b" 的最小 key)
tm.headMap("cherry");        // 小于 cherry 的子视图(视图,改动会反映到原 map)
tm.subMap("apple", "cherry");
for (var e : tm.entrySet()) { }  // 按 key 升序遍历
```

红黑树的五条性质(节点红/黑、根黑、红节点子黑、任一节点到叶黑高相同等)保证最长路径 ≤ 2× 最短,近似平衡。插入/删除靠变色 + 旋转维持,单次调整 O(log n)。

| 对比 | HashMap | TreeMap |
|------|---------|---------|
| 底层 | 数组+链表/红黑树 | 红黑树 |
| 有序 | 无序 | 按 key 排序 |
| get/put | O(1) 均摊 | O(log n) |
| 导航方法 | 无 | floor/ceiling/subMap 等 |
| null key | 允许一个 | 不允许(需比较) |

## Stream API

最简单的用法:对一个 `List` 先筛选、再计数:

```java
List<String> names = List.of("Ann", "Bob", "Al", "Ben");
long count = names.stream()               // 1. 从集合开一条"流水线"
    .filter(s -> s.startsWith("A"))        // 2. 只留下满足条件的元素
    .count();                              // 3. 数一数还剩几个
System.out.println(count);                 // 2
```

`集合.stream()` 开启流水线,中间可以链式接任意多个"加工步骤"(这里是 `filter`),最后用一个"取结果的操作"(这里是 `count`)结束整条链子。记住这个三段式(源 → 中间操作 → 终止操作),下面的内容都是在这个骨架上展开。

Stream 是**声明式、惰性、可并行**的数据处理管道,类比 C++20 ranges/views。不是容器,不存数据,是"对数据源的操作描述"。

### 惰性与操作分类

```java
List<String> words = List.of("apple", "banana", "cherry", "avocado");

long count = words.stream()          // 源
    .filter(s -> s.startsWith("a"))  // 中间操作(intermediate):惰性,返回新 Stream
    .map(String::toUpperCase)        // 中间操作
    .peek(System.out::println)       // 中间操作(调试用)
    .count();                        // 终止操作(terminal):触发实际执行
```

- **中间操作**(`filter`/`map`/`sorted`/`distinct`/`limit`):惰性,只是记录,不执行。返回 Stream 可链式。
- **终止操作**(`collect`/`forEach`/`count`/`reduce`/`findFirst`):触发遍历,执行整条链。

**惰性的意义**:操作**融合**,一个元素流过整条链再取下一个(而非每步都生成完整中间集合),省内存;`limit`/`findFirst` 能**短路**提前结束。

```java
// 短路:找到第一个就停,不会处理整个流
Optional<Integer> first = Stream.iterate(1, n -> n + 1)  // 无限流
    .filter(n -> n % 7 == 0)
    .findFirst();   // Optional[7]
```

### Collectors

终止操作 `collect` 把流收集成结果:

```java
import static java.util.stream.Collectors.*;

List<String> list  = words.stream().collect(toList());
Set<String> set    = words.stream().collect(toSet());
Map<Character, List<String>> byFirst =
    words.stream().collect(groupingBy(s -> s.charAt(0)));   // 分组,类似 SQL GROUP BY
Map<Boolean, List<String>> parts =
    words.stream().collect(partitioningBy(s -> s.length() > 5));
String joined = words.stream().collect(joining(", ", "[", "]"));
Double avg = words.stream().collect(averagingInt(String::length));
```

可运行:`peek` 观察惰性——`findFirst` 短路,只拉到第一个匹配就停,不遍历整个流;`groupingBy` 按首字母分组:

```java
import java.util.*;
import java.util.stream.*;
import static java.util.stream.Collectors.*;

public class StreamDemo {
    public static void main(String[] args) {
        List<String> words = List.of("apple", "banana", "cherry", "avocado", "blueberry");

        // 惰性 + 短路:peek 只会打印真正被拉取的元素
        System.out.println("--- lazy pipeline (findFirst short-circuits) ---");
        Optional<String> first = words.stream()
            .peek(w -> System.out.println("seen: " + w))   // 副作用,用于观察遍历
            .filter(w -> w.startsWith("b"))
            .findFirst();
        System.out.println("first b-word = " + first.get());

        // groupingBy:按首字母分组,用 TreeMap 让输出有序稳定
        Map<Character, List<String>> byFirst = words.stream()
            .collect(groupingBy(w -> w.charAt(0), TreeMap::new, toList()));
        System.out.println("--- groupingBy ---");
        System.out.println(byFirst);
    }
}
// --- lazy pipeline (findFirst short-circuits) ---
// seen: apple
// seen: banana
// first b-word = banana
// --- groupingBy ---
// {a=[apple, avocado], b=[banana, blueberry], c=[cherry]}
```

### 并行流

`parallelStream()` 或 `.parallel()` 把流拆分,底层用**公共 ForkJoinPool**(`ForkJoinPool.commonPool()`,默认线程数 = CPU 核数-1)分治处理。

```java
long sum = LongStream.rangeClosed(1, 1_000_000)
    .parallel()
    .sum();
```

注意事项:
- 数据源要**易拆分**(数组、`ArrayList` 好;`LinkedList` 差)。
- 操作须**无状态、无副作用**,否则数据竞争(不要在 lambda 里改共享变量)。
- 小数据集或廉价操作,并行的拆分/合并/调度开销可能得不偿失。
- 所有并行流共享同一 commonPool,一个慢任务拖累全局;IO 密集别用它。

## Optional

`Optional<T>` 表达"可能没有值",替代裸 `null`,强制调用者处理缺失,减少 NPE。类比 C++17 `std::optional`(但 Java 的是对象,有堆分配开销)。

最基础的用法:`Optional.of` 包一个确定存在的值,`Optional.empty()` 表示确定没有值,`isPresent()` 判断是哪种情况:

```java
Optional<String> opt1 = Optional.of("hello");   // 明确有值
Optional<String> opt2 = Optional.empty();       // 明确没有值
System.out.println(opt1.isPresent());           // true
System.out.println(opt2.isPresent());           // false
```

实际代码里更常见的是用 `Optional.ofNullable` 包一个"可能是 null 的值"(比如一个可能查不到结果的方法返回值),再配合下面这些方法处理"有"或"没有"两种情况,而不必显式写 `if (x != null)`:

```java
Optional<String> opt = Optional.ofNullable(getName());
String s = opt.orElse("default");             // 无值给默认
String t = opt.orElseGet(() -> compute());    // 惰性默认(仅无值时算)
opt.ifPresent(System.out::println);
String u = opt.map(String::trim).filter(x -> !x.isEmpty()).orElse("-");
opt.orElseThrow(() -> new IllegalStateException("missing"));
```

用法约定:主要用作**方法返回值**表达"可能无结果";**不要**用于字段、方法参数或集合元素(徒增开销)。

## 数值工具

日常用 `Math` 的静态方法做基本运算;当 `int`/`long`/`double` 精度或范围不够用时(比如需要超过 64 位的整数,或金融计算不能有浮点误差),再用 `BigInteger`/`BigDecimal`。

```java
Math.max(a, b); Math.floorMod(-1, 3); // 2(与 % 不同,结果符号随除数,恒为非负)
Math.addExact(a, b);                  // 溢出抛 ArithmeticException,而非静默回绕

// BigInteger:任意精度整数(不可变),类比自己实现的大数;运算返回新对象
BigInteger big = BigInteger.valueOf(2).pow(1000);
BigInteger f = big.multiply(BigInteger.TEN).mod(BigInteger.valueOf(7));

// BigDecimal:精确十进制,金融必用(double 无法精确表示 0.1)
BigDecimal price = new BigDecimal("0.10");    // 必须用 String 构造,new BigDecimal(0.1) 会带二进制误差
BigDecimal total = price.multiply(new BigDecimal("3"));  // 精确 0.30
System.out.println(new BigDecimal("0.1").add(new BigDecimal("0.2"))); // 0.3(double 会是 0.30000000000000004)
total = total.setScale(2, RoundingMode.HALF_UP);         // 显式指定舍入

// Random:线性同余伪随机;并发用 ThreadLocalRandom;安全场景用 SecureRandom
int r = ThreadLocalRandom.current().nextInt(0, 100);
```

> `BigInteger`/`BigDecimal` 不可变,频繁运算产生大量临时对象,循环中注意 GC 压力。它们的比较用 `compareTo`,`BigDecimal` 的 `equals` 还会比较 scale(`2.0` != `2.00`),用 `compareTo == 0` 判数值相等。

## 小结

- **用法层面**:
  - `Collections` 是操作 `List`/`Collection` 的静态方法集合:`sort`/`binarySearch`/`reverse`/`shuffle`/`max`;不可变视图用 `unmodifiableXxx`(包装原集合)或 `List.of`(真正的不可变副本)。
  - `Arrays.sort` 对基本类型数组和对象数组调用方式一样,内部自动选算法。
  - 给对象排序:类自己实现 `Comparable`(只有一种自然顺序),或传入外部的 `Comparator`(可以有任意多种),JDK8+ 用 `Comparator.comparing().thenComparing().reversed()` 声明式组合。
  - `PriorityQueue` 就是堆,默认小顶堆,`offer`/`peek`/`poll` 增删查堆顶。
  - `TreeMap`/`TreeSet` 按 key 天然有序,多了 `firstKey`/`floorKey`/`ceilingKey`/`headMap`/`subMap` 等 `HashMap` 没有的导航方法。
  - Stream 的骨架是"源 `.stream()` → 若干中间操作(`filter`/`map`/`sorted`)→ 一个终止操作(`collect`/`count`/`forEach`)";`collect` 配合 `Collectors` 做分组、分区、拼接、统计。
  - `Optional<T>` 用来表达"可能没有值",主要当方法返回值用,配合 `orElse`/`map`/`ifPresent` 处理,不要用在字段或参数上。
  - 超出 `long` 范围用 `BigInteger`,需要精确十进制(金融场景)用 `BigDecimal`——`BigDecimal` 必须用字符串构造,比较数值相等要用 `compareTo == 0` 而不是 `equals`。
- **底层层面**:
  - `Arrays.sort`:基本类型用**双轴快排**(相等的值本就无差别,稳定性无意义,原地、缓存友好);对象用 **TimSort**(归并+插入的混合,识别已排序的 run,保证稳定,最坏 O(n log n)、已有序数据 O(n),需要 O(n) 辅助空间)。
  - `Comparator`/`Comparable` 的契约要求自反、传递、反对称,违反传递性会让 TimSort 直接抛 `IllegalArgumentException`。
  - `PriorityQueue` 底层是**数组表示的二叉堆**(节点 `i` 的孩子是 `2i+1`/`2i+2`),`offer` 上浮、`poll` 下沉,均为 O(log n),非线程安全。
  - `TreeMap` 底层是**红黑树**,靠"根黑、红节点子必黑、任一路径黑高相同"等性质保证近似平衡,所有操作 O(log n)。
  - Stream 的中间操作全部**惰性**,直到终止操作才真正执行,好处是操作能融合(省内存)且能**短路**(`limit`/`findFirst` 提前结束);并行流靠公共 `ForkJoinPool` 分治,要求数据源易拆分、操作无状态无副作用。
  - `BigInteger`/`BigDecimal` 都是**不可变**对象,每次运算生成新对象,循环里大量使用要留意 GC 压力。
